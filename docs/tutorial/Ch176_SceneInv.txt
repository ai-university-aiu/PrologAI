Chapter 176: sceneinv Pack - Scene Invariant Detection (si_*, Layer 194)


Overview

The sceneinv pack analyses lists of Before-After training pairs to detect
structural properties that hold across ALL examples. Knowing what stays
constant gives you constraints that any correct solution must satisfy.

Layer 194. Pack prefix: si_*. Work Package 215.

A pair is Before-After where Before and After are lists of obj(Color, Cells) terms.


The Three Invariant Families

Object-count invariants: does the number of objects stay fixed?

Color-set invariants: does the set of distinct colors stay the same?

Cell-count invariants: does the total number of cells stay the same?


Exported Predicates


si_n_before(+Pair, -N)

N is the number of objects in the Before scene of Pair.

Example:

si_n_before([obj(r,[r(0,0)]),obj(b,[r(0,1)])]-[obj(r,[r(0,0)])], N).
% N = 2


si_n_after(+Pair, -N)

N is the number of objects in the After scene of Pair.


si_all_n_before(+Pairs, -Ns)

Ns is the list of Before object counts, one per pair.

Example:

Pairs = [[obj(r,[r(0,0)])]-[obj(b,[r(0,0)])],
         [obj(r,[r(0,0)])]-[obj(b,[r(0,0)])]],
si_all_n_before(Pairs, Ns).
% Ns = [1, 1]


si_all_n_after(+Pairs, -Ns)

Ns is the list of After object counts, one per pair.


si_const_n_before(+Pairs, -N)

N is constant across all Before scenes. Fails if counts differ or Pairs is empty.

Example:

si_const_n_before(Pairs, N).
% N = 1  (if every Before has exactly 1 object)


si_const_n_after(+Pairs, -N)

N is constant across all After scenes. Fails if counts differ or Pairs is empty.


si_n_preserved(+Pairs)

Succeeds if Before and After have the same object count in EVERY pair.
Succeeds vacuously for empty Pairs.

Example:

si_n_preserved([B1-A1, B2-A2]).
% succeeds if length(B1)==length(A1) and length(B2)==length(A2)


si_colors_before(+Pair, -Colors)

Sorted distinct colors in the Before scene. Uses sort/2 so duplicates are removed.

Example:

si_colors_before([obj(r,[r(0,0)]),obj(b,[r(0,1)])]-_, Colors).
% Colors = [b, r]


si_colors_after(+Pair, -Colors)

Sorted distinct colors in the After scene.


si_const_colors_before(+Pairs, -Colors)

Colors is the Before color set, constant across all pairs.
Fails if sets differ or Pairs is empty.


si_const_colors_after(+Pairs, -Colors)

Colors is the After color set, constant across all pairs.
Fails if sets differ or Pairs is empty.


si_colors_preserved(+Pairs)

Succeeds if Before and After have the same sorted color set in EVERY pair.
Succeeds vacuously for empty Pairs.

Example:

% Color-swap pair: {r,b} -> {r,b}  -- preserved
Pair = [obj(r,[r(0,0)]),obj(b,[r(0,1)])] - [obj(b,[r(0,0)]),obj(r,[r(0,1)])],
si_colors_preserved([Pair]).  % succeeds


si_total_cells_before(+Pair, -N)

Total cell count across all objects in the Before scene.
N = sum of length(Cells) for each obj(_, Cells) in Before.

Example:

si_total_cells_before([obj(r,[r(0,0),r(0,1)])]-_, N).
% N = 2


si_cells_preserved(+Pairs)

Succeeds if the total cell count is the same in Before and After for EVERY pair.
Succeeds vacuously for empty Pairs.


Typical Use

Given a list of training pairs, check invariants before searching for a rule:

solve(Pairs, Test, Output) :-
    ( si_n_preserved(Pairs) ->
        % output must have same object count as Test
        length(Test, N), length(Output, N)
    ; true ),
    % ... continue with rule search


Dependencies

Summary

The sceneinv pack provides a principled way to extract invariants from training
data before committing to a transformation hypothesis. Combined with gridsolve
and scenerank, it constrains the solution space and can reject candidate rules
that violate structural properties common to all training examples.
