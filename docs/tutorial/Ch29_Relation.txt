PROLOGAI TUTORIAL
Chapter 29: The Relation Pack


OVERVIEW

The relation pack (module relation, prefix rl_) provides spatial relations
between cell regions. It lives in Layer 47 of the PrologAI cognitive stack
and depends only on the grid pack.

A region is any list of r(R,C) cells. Regions do not need to be normalized
to the origin. The relation pack answers questions like "is this object above
that object?", "how far apart are these regions?", "which direction is B
from A?".

Predicates: 14
Layer: 47
Module: relation
Prefix: rl_
Depends on: grid


LOADING THE RELATION PACK

:- use_module(library(grid)).
:- use_module(library(relation)).


POSITIONAL ORDERING

relation_above(+A, +B)

Succeed if every row of A is strictly less than every row of B.
(In grid coordinates, row 0 is at the top, so smaller row = higher up.)

% A 2-row block at rows 0-1 is above a 2-row block at rows 3-4.
:- relation_above([r(0,0),r(1,0)], [r(3,0),r(4,0)]).
% Succeeds.

:- relation_above([r(3,0)], [r(0,0)]).
% Fails.

relation_below(+A, +B)

Succeed if every row of A is strictly greater than every row of B.

:- relation_below([r(3,0),r(4,0)], [r(0,0),r(1,0)]).
% Succeeds.

relation_left_of(+A, +B)

Succeed if every column of A is strictly less than every column of B.

:- relation_left_of([r(0,0),r(0,1)], [r(0,3),r(0,4)]).
% Succeeds.

relation_right_of(+A, +B)

Succeed if every column of A is strictly greater than every column of B.

:- relation_right_of([r(0,3),r(0,4)], [r(0,0),r(0,1)]).
% Succeeds.


ADJACENCY AND DISTANCE

relation_adjacent(+A, +B)

Succeed if any cell of A has a 4-connected neighbor in B.

% Two regions touching at a border.
:- relation_adjacent([r(0,0),r(1,0)], [r(2,0)]).
% Succeeds.

% Two regions separated by a gap.
:- relation_adjacent([r(0,0)], [r(2,0)]).
% Fails.

relation_distance(+A, +B, -D)

D is the minimum Manhattan distance between any cell of A and any cell of B.
Returns 0 if A and B share a cell.

% Adjacent single cells.
:- relation_distance([r(0,0)], [r(0,1)], D).
% D = 1.

% Overlapping regions.
:- relation_distance([r(1,1),r(1,2)], [r(1,2),r(1,3)], D).
% D = 0.


CONTAINMENT AND SET RELATIONS

relation_contained_bbox(+A, +B)

Succeed if the bounding box of A is contained within the bounding box of B.

% A single inner cell inside a surrounding ring.
:- relation_contained_bbox([r(1,1)],
                      [r(0,0),r(0,1),r(0,2),r(1,0),r(1,2),r(2,0),r(2,1),r(2,2)]).
% Succeeds.

relation_overlap(+A, +B)

Succeed if A and B share at least one cell.

:- relation_overlap([r(0,0),r(0,1)], [r(0,1),r(0,2)]).
% Succeeds.

:- relation_overlap([r(0,0)], [r(1,0)]).
% Fails.

relation_disjoint(+A, +B)

Succeed if A and B share no cells.

:- relation_disjoint([r(0,0)], [r(1,0)]).
% Succeeds.


ROW AND COLUMN ALIGNMENT

relation_same_row(+A, +B)

Succeed if A and B have at least one row index in common.

% Two regions sharing row 2.
:- relation_same_row([r(2,0),r(2,1)], [r(2,5)]).
% Succeeds.

relation_same_col(+A, +B)

Succeed if A and B have at least one column index in common.

% Two regions sharing column 3.
:- relation_same_col([r(0,3),r(1,3)], [r(5,3)]).
% Succeeds.


CENTROID, OFFSET, AND DIRECTION

relation_centroid(+Region, -AvgR, -AvgC)

AvgR and AvgC are the integer (floor division) centroid of Region.

% Centroid of a 3-cell vertical bar at rows 0,1,2; col 2.
:- relation_centroid([r(0,2),r(1,2),r(2,2)], R, C).
% R = 1, C = 2.

relation_offset(+A, +B, -DR, -DC)

DR = centroid_row(B) - centroid_row(A).
DC = centroid_col(B) - centroid_col(A).

% B is 3 rows below and 4 cols to the right of A.
:- relation_offset([r(0,0)], [r(3,4)], DR, DC).
% DR = 3, DC = 4.

relation_direction(+A, +B, -Dir)

Dir is the cardinal direction from A toward B: above, below, left, or right.
The dominant axis of the centroid offset determines the direction.

% B is above A.
:- relation_direction([r(5,5)], [r(2,5)], Dir).
% Dir = above.

% B is to the right of A.
:- relation_direction([r(2,1)], [r(2,5)], Dir).
% Dir = right.

% Diagonal: row-dominant becomes above.
:- relation_direction([r(5,0)], [r(2,1)], Dir).
% Dir = above (DR = -3, DC = 1; |DR| >= |DC|).


TYPICAL USAGE PATTERNS

Testing if a small object fits inside a larger frame:

contains_region(Outer, Inner) :-
    relation_contained_bbox(Inner, Outer),
    relation_disjoint(Inner, Outer).

Finding which region is nearest to a target:

nearest_region(Target, Regions, Nearest) :-
    findall(D-R,
        (member(R, Regions),
         relation_distance(Target, R, D)),
        Pairs),
    min_member(_-Nearest, Pairs).

Building a spatial layout description:

describe_layout(A, B, Desc) :-
    (relation_above(A, B) -> Desc = a_above_b
    ;relation_below(A, B) -> Desc = a_below_b
    ;relation_left_of(A, B) -> Desc = a_left_of_b
    ;relation_right_of(A, B) -> Desc = a_right_of_b
    ;relation_adjacent(A, B) -> Desc = a_adjacent_b
    ;relation_overlap(A, B) -> Desc = a_overlaps_b
    ;Desc = a_disjoint_b
    ).


PREDICATE SUMMARY

Positional ordering:
  relation_above/2      - A entirely above B
  relation_below/2      - A entirely below B
  relation_left_of/2    - A entirely left of B
  relation_right_of/2   - A entirely right of B

Adjacency and distance:
  relation_adjacent/2   - 4-connected adjacency
  relation_distance/3   - minimum Manhattan distance

Containment and set relations:
  relation_contained_bbox/2 - bounding box containment
  relation_overlap/2        - shared cell existence
  relation_disjoint/2       - no shared cells

Row and column alignment:
  relation_same_row/2   - shared row index
  relation_same_col/2   - shared column index

Centroid and direction:
  relation_centroid/3   - integer centroid
  relation_offset/4     - integer centroid offset (DR, DC)
  relation_direction/3  - cardinal direction
