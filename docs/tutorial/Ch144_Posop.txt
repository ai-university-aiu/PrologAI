Chapter 144: The Posop Pack - Position-Based Sorting, Filtering, and Assignment

The posop pack (po_* prefix, Layer 162) provides 14 predicates that work with the
spatial position of obj(Color, Cells) terms in a scene. "Position" means where an
object sits in the grid: specifically, its topmost row (the smallest row index in
its Cells list) and its leftmost column (the smallest column index).

This chapter explains all 14 predicates with examples.


Section 144.1: What Is "Position"?

Every obj(Color, Cells) term has a list of r(Row, Col) cell coordinates. The
"position" of an object is defined by:

- Topmost row: the minimum Row value across all cells.
- Leftmost col: the minimum Col value across all cells.

Example: obj(red, [r(2,3), r(2,4), r(3,3)]) has topmost row 2 and leftmost col 3.

These two values together form the object's "position anchor" - the top-left corner
of its bounding box.


Section 144.2: Position Extraction

po_row_of(+Obj, -Row)

Row is the topmost (minimum) row index of Obj's cells.

    ?- po_row_of(obj(red, [r(2,3), r(2,4), r(3,3)]), Row).
    Row = 2.

po_col_of(+Obj, -Col)

Col is the leftmost (minimum) column index of Obj's cells.

    ?- po_col_of(obj(red, [r(2,3), r(2,4), r(3,3)]), Col).
    Col = 3.


Section 144.3: Position Rank Queries

These three predicates tell you where an object ranks among others by position.

po_row_rank(+Objs, +Obj, -Rank)

Rank is the 1-based position of Obj when Objs is sorted by topmost row ascending.
Rank 1 = topmost object. Ties (equal rows) are broken by the original input order.

    Objs = [obj(a,[r(4,0)]), obj(b,[r(0,0)]), obj(c,[r(2,0)])],
    po_row_rank(Objs, obj(b,[r(0,0)]), Rank).
    Rank = 1.   % obj(b) has the smallest row (0), so it ranks first.

po_col_rank(+Objs, +Obj, -Rank)

Rank is the 1-based position of Obj when Objs is sorted by leftmost col ascending.
Rank 1 = leftmost object.

po_reading_rank(+Objs, +Obj, -Rank)

Rank is the 1-based position of Obj in reading order: ascending by topmost row,
then by leftmost col within equal rows.

    % For objects at (0,0), (0,3), (2,0):
    % Reading order: (0,0) first, (0,3) second, (2,0) third.
    po_reading_rank(Objs, obj_at_2_0, Rank).
    Rank = 3.


Section 144.4: Color Assignment by Position

These predicates sort objects by position and then assign colors from a list.
They are the positional complement of sz_assign_colors/3 in the sizeop pack.

po_assign_by_row(+Objs, +Colors, -Result)

Sorts Objs by topmost row ascending, then assigns colors in that order. The
topmost object gets Colors[1], the next gets Colors[2], and so on. If Colors is
shorter than Objs, only that many objects are recolored. If Objs is shorter,
only that many colors are used.

    Objs = [obj(green,[r(3,0)]), obj(red,[r(0,0)]), obj(blue,[r(1,0)])],
    po_assign_by_row(Objs, [top_c, mid_c, bot_c], Result).
    % Sort by row: r0->red, r1->blue, r3->green.
    Result = [obj(top_c,[r(0,0)]), obj(mid_c,[r(1,0)]), obj(bot_c,[r(3,0)])].

po_assign_by_col(+Objs, +Colors, -Result)

Sorts Objs by leftmost col ascending, then assigns colors in that order.

po_assign_reading(+Objs, +Colors, -Result)

Sorts Objs in reading order (row then col), then assigns colors in that order.

    Objs = [obj(a,[r(0,3)]), obj(b,[r(0,0)]), obj(c,[r(1,0)])],
    po_assign_reading(Objs, [first, second, third], Result).
    % Reading order: (0,0)->b, (0,3)->a, (1,0)->c.
    Result = [obj(first,[r(0,0)]), obj(second,[r(0,3)]), obj(third,[r(1,0)])].


Section 144.5: Threshold Filtering

These predicates split an object list into two groups based on a row or column
boundary. Each predicate returns one of the two groups.

po_above_row(+Objs, +R, -Filtered)

Returns objects whose topmost row is strictly less than R.

    % For objects at rows 0, 2, 4 in a scene:
    po_above_row(Objs, 2, Filtered).
    % Filtered contains only the row-0 object.

po_from_row(+Objs, +R, -Filtered)

Returns objects whose topmost row is >= R (at row R or below it).

    po_from_row(Objs, 2, Filtered).
    % Filtered contains row-2 and row-4 objects.

po_left_of(+Objs, +C, -Filtered)

Returns objects whose leftmost col is strictly less than C.

po_from_col(+Objs, +C, -Filtered)

Returns objects whose leftmost col is >= C.

Together, po_above_row and po_from_row split the scene at a horizontal boundary.
po_left_of and po_from_col split it at a vertical boundary.


Section 144.6: Band Filtering

Band filtering keeps objects within a range of rows or columns.

po_in_row_band(+Objs, +R1, +R2, -Filtered)

Returns objects whose topmost row is in the inclusive range [R1, R2].

    % Objects at rows 0, 2, 4. Band [0, 2]:
    po_in_row_band(Objs, 0, 2, Filtered).
    % Filtered: row-0 and row-2 objects (row 4 excluded).

po_in_col_band(+Objs, +C1, +C2, -Filtered)

Returns objects whose leftmost col is in the inclusive range [C1, C2].


Section 144.7: How Reading Order Sort Works

The reading-order sort (used internally by po_sort_reading_ and exposed via
po_reading_rank and po_assign_reading) uses a compound key Row-Col as the
keysort/2 key. In SWI-Prolog's standard order of terms, the -/2 functor sorts
first by its left argument (Row) then by its right argument (Col) when left
arguments are equal. This gives row-then-column ordering - exactly reading order.

keysort/2 is stable: equal compound keys (same row AND same col) preserve input
order. This is only possible if two objects occupy the exact same anchor position,
which cannot happen for distinct objects in a valid scene.


Section 144.8: Complementary Packs

posop and sizeop (sz_*, Layer 161) are the two "ranking" packs:

- sizeop ranks objects by CELL COUNT (how large they are).
- posop ranks objects by SPATIAL POSITION (where they are).

Both offer color assignment by rank (sz_assign_colors, po_assign_by_row, etc.)
and selection predicates (sz_smallest, po_above_row, etc.). They are independent
and can be combined in composite rules.

The order pack (od_*, earlier layer) sorts objects by position but does not
provide threshold filtering or color assignment in position order. posop fills
this gap.


Section 144.9: Quick Reference

Position extraction:
- po_row_of/2      topmost row of an object
- po_col_of/2      leftmost col of an object

Rank queries:
- po_row_rank/3    1-based rank by topmost row (1 = topmost)
- po_col_rank/3    1-based rank by leftmost col (1 = leftmost)
- po_reading_rank/3 1-based rank in reading order

Color assignment by position:
- po_assign_by_row/3   topmost-first assignment
- po_assign_by_col/3   leftmost-first assignment
- po_assign_reading/3  reading-order assignment

Threshold filtering:
- po_above_row/3   row < R
- po_from_row/3    row >= R
- po_left_of/3     col < C
- po_from_col/3    col >= C

Band filtering:
- po_in_row_band/4 row in [R1, R2]
- po_in_col_band/4 col in [C1, C2]
