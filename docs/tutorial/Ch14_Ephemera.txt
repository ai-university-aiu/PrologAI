CHAPTER 14 - THE EPHEMERA PACK: WRITING AND RUNNING SHORT-LIVED PROGRAMS

The ephemera pack gives PrologAI the ability to compose a small program, run it, capture its output, and then discard the program entirely.

These short-lived programs are called ephemera (singular: ephemeron).

They are created for a purpose, used once, and then gone - just like a scratch calculation you write on a piece of paper and then throw away.

The name comes from the Greek word for "lasting only a day."


WHY EPHEMERA EXIST

Symbolic logic in Prolog is excellent for structured reasoning, but some tasks are more naturally expressed as a small, runnable program than as a set of logical facts.

For example:
- Converting between data formats
- Performing a numerical calculation too complex for arithmetic/1
- Calling a system tool via a shell command
- Quickly testing whether a hypothesis holds as executable code

Capable problem-solving systems - both skilled human programmers and modern AI coding agents - routinely write short throwaway programs to probe unknowns, test hypotheses, and automate mechanical steps.

The ephemera pack gives PrologAI the same capability.


LOADING THE EPHEMERA PACK

Load the pack in your session or your program file:

    :- use_module(library(ephemera)).

This makes all ep_* predicates available.


EVALUATING A PROLOG GOAL WITH A TIMEOUT

The simplest ephemeron is a Prolog goal evaluated with a time limit.

ephemera_eval(+GoalTerm, +TimeoutSecs, -EvalResult)

Example:

    ?- ephemera_eval((X is 3 * 7), 5, R).
    X = 21,
    R = success.

The result can be one of four values:

    success     The goal succeeded and any variables in GoalTerm are now bound.

    fail        The goal deterministically failed (no solution exists).

    timeout     The timeout expired before the goal finished.

    error(E)    The goal threw the exception E.

The variable bindings in GoalTerm are preserved after a success result, so you can extract computed values from the original goal term directly after ephemera_eval/3 returns.


EVALUATING PROLOG CODE WRITTEN AS AN ATOM

Sometimes the code to evaluate is built up as a text atom - for example, when it is constructed from parts at runtime.

ephemera_scratch(+CodeAtom, +VarBindings, +TimeoutSecs, -EvalResult)

CodeAtom is a Prolog goal written as a text atom, for example 'X is 2 + 2'.

VarBindings is a list of pairs 'VarName'=Variable, where VarName is the name as it appears in the code atom.

Example:

    ?- ephemera_scratch('Y is 10 + 5', ['Y'=Y], 5, R).
    Y = 15,
    R = success(['Y'=15]).

The result on success is success(UpdatedBindings), where UpdatedBindings is the same list with the variable values filled in.


RUNNING A SHELL COMMAND

ephemera_shell(+Argv, +TimeoutSecs, -ShellResult)

Argv is a list of atoms.

The first atom is the executable.

The remaining atoms are its arguments.

Example:

    ?- ephemera_shell(['echo', 'hello world'], 10, R).
    R = shell_result(0, 'hello world\n', '').

The result is always a shell_result/3 term:

    shell_result(ExitCode, Stdout, Stderr)

ExitCode is 0 on success.

Stdout and Stderr are atoms containing the captured output from each stream.

You can run any command your operating system supports:

    ?- ephemera_shell(['date'], 5, shell_result(_, Out, _)).
    Out = 'Tue Jun 24 12:00:00 UTC 2026\n'.

    ?- ephemera_shell(['python3', '--version'], 5, shell_result(_, Out, Err)).
    Out = '',
    Err = 'Python 3.12.0\n'.


RUNNING A SCRIPT IN ANOTHER LANGUAGE

ephemera_ephemeral(+Language, +Code, +TimeoutSecs, -ShellResult)

Language is one of: prolog, python, or bash.

Code is the source code of the script as an atom.

The pack writes the code to a temporary file, runs it, captures output, then deletes the temporary file.

Example - run a Python snippet:

    ?- ephemera_ephemeral(python,
           'import math\nprint(math.sqrt(144))',
           10, R).
    R = shell_result(0, '12.0\n', '').

Example - run a Bash snippet:

    ?- ephemera_ephemeral(bash,
           'echo "files: $(ls | wc -l)"',
           10, shell_result(0, Out, _)).

Example - run Prolog source code as a script:

    ?- ephemera_ephemeral(prolog,
           ':- write(hello_from_script), nl, halt.',
           10, shell_result(0, Out, _)).
    Out = 'hello_from_script\n'.

The temporary file is always deleted, even if the script fails or times out.


ITERATION CYCLES: SYNTHESIZE, EXECUTE, CHECK

ephemera_iterate(:SynthGoal, :ExecGoal, :CheckGoal, +MaxIter, -IterResult)

This predicate implements a closed loop:

1. Call SynthGoal to produce Code.
2. Call ExecGoal with Code to produce Result.
3. Call CheckGoal with Result.
4. If CheckGoal succeeds, the loop ends with ok(Code, Result).
5. If CheckGoal fails and the budget remains, go to step 1.
6. If the budget is exhausted, end with exhausted(Code, Result).

This is the building block for code-generation loops: the synthesizer proposes code, the executor runs it, the checker decides whether the result is good enough.

Example pattern:

    synth([C]>>(C = 'print(42)')).
    run([C, R]>>(ephemera_ephemeral(python, C, 5, R))).
    check([R]>>(R = shell_result(0, _, _))).

    ?- ephemera_iterate(synth, run, check, 5, Result).
    Result = ok('print(42)', shell_result(0, '42\n', '')).


TRACING EXECUTION SEQUENCES

When you want to review a sequence of ephemeron executions later, use the trace log.

Allocate a trace identifier:

    ?- ephemera_next_trace_id(TId).
    TId = ep_trace_1.

Record each step:

    ?- ephemera_trace_record(TId, 1, 'X is 1', success).
    ?- ephemera_trace_record(TId, 2, 'Y is 2', success).

Retrieve the full trace:

    ?- ephemera_trace_get(TId, Entries).
    Entries = [trace(TId, 1, 'X is 1', success),
               trace(TId, 2, 'Y is 2', success)].

The trace ID is unique across all trace sessions in the current process.

Each entry records: the trace ID, the step number, the code that was run, and the output or result.


SAFETY PROPERTIES

Every subprocess execution is bounded by the caller-supplied TimeoutSecs.

If a subprocess does not finish in time, it is terminated and a shell_result with exit code -1 is returned.

Temporary files created by ephemera_ephemeral/4 are deleted before the predicate returns, on both success and failure.

The ephemera pack does not modify any global state except its own trace log and counter.

All external code (Python, Bash, external Prolog) runs in a subprocess isolated from the main PrologAI process.


SKILL PERSISTENCE: NAMING AND REUSING USEFUL EPHEMERA

An ephemeron that proves useful during execution does not have to stay ephemeral.

If the same computation is likely to be needed again, it can be saved as a named skill and retrieved later without re-synthesis.

SAVING A SKILL

ephemera_skill_save/4 takes four arguments: a name atom, a language atom, the source code atom, and a human-readable description.

% Save a greeting skill in Python with a description.
ephemera_skill_save(greet, python, 'print("Hello from Mentova")',
              'Print a greeting from Mentova').

If a skill with the same name already exists it is replaced.


RETRIEVING A SKILL

ephemera_skill_lookup/3 retrieves the language and code for a named skill.

% Look up the greeting skill.
ephemera_skill_lookup(greet, Lang, Code).
% Lang = python
% Code = 'print("Hello from Mentova")'

It fails silently if no skill with that name exists.


RUNNING A SAVED SKILL

ephemera_skill_run/3 retrieves a skill by name and executes it immediately.

% Run the greeting skill with a 10-second timeout.
ephemera_skill_run(greet, 10, shell_result(Code, Out, _)).
% Code = 0
% Out  = 'Hello from Mentova\n'

This is exactly equivalent to ephemera_skill_lookup/3 followed by ephemera_ephemeral/4 - ephemera_skill_run/3 is a thin named-dispatch convenience.


LISTING ALL SAVED SKILLS

ephemera_skill_list/1 returns all saved skills as a list of skill(Name, Language, Description) terms.

The code body is not included in the listing - this keeps the list compact and readable.

% List all saved skills.
ephemera_skill_list(Skills).
% Skills = [skill(greet, python, 'Print a greeting from Mentova')]

An empty list is returned when no skills have been saved.


FORGETTING A SKILL

ephemera_skill_forget/1 removes a saved skill by name.

% Remove the greeting skill.
ephemera_skill_forget(greet).

It succeeds silently even if no skill with that name exists.


WHEN TO USE SKILL PERSISTENCE

Save an ephemeron as a skill when:
- The same computation is needed in multiple places in the same session.
- A synthesized ephemeron has been verified correct and should not be re-synthesized each time.
- You want to build an index of reusable utility code that grows with the session.

Skills persist for the lifetime of the SWI-Prolog session. They are stored in the dynamic fact ephemera_skill_db/4, which is not written to disk. To persist skills across sessions, export ephemera_skill_db/4 facts to a file and reload them at startup.


SAFETY PROPERTIES

Every subprocess execution is bounded by the caller-supplied TimeoutSecs.

If a subprocess does not finish in time, it is terminated and a shell_result with exit code -1 is returned.

Temporary files created by ephemera_ephemeral/4 are deleted before the predicate returns, on both success and failure.

The ephemera pack does not modify any global state except its own trace log, counter, and skill database.

All external code (Python, Bash, external Prolog) runs in a subprocess isolated from the main PrologAI process.


SUMMARY

ephemera_eval/3             Evaluate a Prolog goal term with timeout.
ephemera_scratch/4          Evaluate Prolog source written as an atom.
ephemera_shell/3            Run any shell command and capture output.
ephemera_ephemeral/4        Write a script to a temp file, run it, delete it.
ephemera_iterate/5          Synthesize-execute-check loop with step budget.
ephemera_trace_record/4     Record one step to a trace log.
ephemera_trace_get/2        Retrieve all entries for a trace.
ephemera_next_trace_id/1    Allocate a fresh unique trace identifier.
ephemera_skill_save/4       Save a useful ephemeron as a named, reusable skill.
ephemera_skill_lookup/3     Retrieve a saved skill's language and code by name.
ephemera_skill_run/3        Execute a saved skill by name.
ephemera_skill_list/1       List all saved skills.
ephemera_skill_forget/1     Remove a saved skill by name.
