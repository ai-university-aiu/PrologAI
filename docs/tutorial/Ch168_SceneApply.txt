Chapter 168: sceneapply Pack - Scene-Level Rule Term Evaluation Engine (sa_*, Layer 186)


Overview

The sceneapply pack applies symbolic rule terms to obj(Color, Cells) scene lists.
It is the evaluation complement to the ruleinfer pack (ri_*): where ruleinfer
infers which rule was applied from example pairs, sceneapply executes that rule
on any scene.

Together these two packs form a complete learn-and-apply cycle: infer the rule
from training examples, verify it across all pairs, then apply it to the test
input.

Layer 186. Pack prefix: sa_*. Work Package 207.


Rule Term Vocabulary

A rule term is a Prolog term with one of these forms:

recolor(Old, New)       - replace objects of Old color with New
recolor_all(New)        - set all objects to New
color_map(Map)          - apply Old-New substitution pairs
shift(DR, DC)           - translate all objects by (DR, DC)
to_origin               - shift scene so bounding box is at r(0,0)
reflect_h(Width)        - flip all objects horizontally
reflect_v(Height)       - flip all objects vertically
remove_color(Color)     - discard all objects of Color
keep_color(Color)       - keep only objects of Color
sorting_size_desc          - sort by cell count descending
sorting_size_asc           - sort by cell count ascending
sorting_pos                - sort by top-left position row-major
top_n(N)                - the N largest objects
dedup_form              - keep first of each distinct normalized shape


Loading the Pack

:- use_module(pack(sceneapply/prolog/scene_apply)).


Core Predicates

scene_apply_apply(+Rule, +Scene, -Scene2)

Apply one rule term to the scene. Dispatches internally to the appropriate
scene manipulation helper based on the rule term structure.

  :- scene_apply_apply(recolor(r,g), [obj(r,[r(0,0)]), obj(b,[r(0,1)])], Out).
  Out = [obj(g,[r(0,0)]), obj(b,[r(0,1)])].

  :- scene_apply_apply(shift(2,3), [obj(r,[r(0,0)])], Out).
  Out = [obj(r,[r(2,3)])].

  :- scene_apply_apply(keep_color(r), [obj(r,[r(0,0)]), obj(b,[r(0,1)]), obj(r,[r(0,2)])], Out).
  Out = [obj(r,[r(0,0)]), obj(r,[r(0,2)])].

scene_apply_apply_seq(+Rules, +Scene, -Scene2)

Apply a sequence of rule terms in order. The first rule transforms the scene,
the second transforms the result, and so on.

  :- scene_apply_apply_seq([recolor(r,g), shift(1,1)], [obj(r,[r(0,0)])], Out).
  Out = [obj(g,[r(1,1)])].

  :- scene_apply_apply_seq([], Scene, Out).
  Out = Scene.  % empty sequence is identity

scene_apply_verify(+Rule, +Before, +After)

Succeed if applying Rule to Before produces After. Comparison is
order-insensitive (uses msort internally).

  :- scene_apply_verify(recolor(r,g), [obj(r,[r(0,0)])], [obj(g,[r(0,0)])]).
  true.

scene_apply_verify_seq(+Rules, +Before, +After)

Succeed if applying the rule sequence to Before produces After.

scene_apply_verify_all(+Rule, +Pairs)

Succeed if Rule works on every Before-After pair. Pairs is a list of
Before-After terms.

  :- P1 = [obj(r,[r(0,0)])] - [obj(g,[r(0,0)])],
     P2 = [obj(r,[r(1,1)])] - [obj(g,[r(1,1)])],
     scene_apply_verify_all(recolor(r,g), [P1, P2]).
  true.

scene_apply_verify_seq_all(+Rules, +Pairs)

Succeed if the rule sequence works on every Before-After pair.


Classification Predicates

scene_apply_rule_type(+Rule, -Type)

Classify a rule term. Returns one of: color, spatial, filter, order, dedup.

  :- scene_apply_rule_type(recolor(r,g), T).   T = color.
  :- scene_apply_rule_type(shift(1,2), T).     T = spatial.
  :- scene_apply_rule_type(keep_color(r), T).  T = filter.
  :- scene_apply_rule_type(sorting_size_desc, T). T = order.
  :- scene_apply_rule_type(dedup_form, T).     T = dedup.

scene_apply_is_color_rule(+Rule)
scene_apply_is_spatial_rule(+Rule)
scene_apply_is_filter_rule(+Rule)

Type-check predicates that delegate to scene_apply_rule_type.

scene_apply_colors_affected(+Rule, -Colors)

List the color atoms referenced by the rule. Spatial, ordering, and dedup rules
return []. Color rules return the colors they operate on.

  :- scene_apply_colors_affected(recolor(r,g), Colors).   Colors = [r,g].
  :- scene_apply_colors_affected(shift(2,3), Colors).     Colors = [].

scene_apply_rule_invertible(+Rule)

Succeed if the rule has a trivially constructible inverse:
- recolor(Old, New) is invertible via recolor(New, Old)
- shift(DR, DC) is invertible via shift(-DR, -DC)
- reflect_h(Width) is self-inverse
- reflect_v(Height) is self-inverse

Filter rules (keep_color, remove_color), ordering rules, and dedup_form are not
trivially invertible.


Composition Utilities

scene_apply_compose(+Rule1, +Rule2, -Seq)

Make a two-step sequence from two rules.

  :- scene_apply_compose(recolor(r,g), shift(1,1), Seq).
  Seq = [recolor(r,g), shift(1,1)].

scene_apply_seq_len(+Seq, -N)

Count the number of steps in a rule sequence.

  :- scene_apply_seq_len([recolor(r,g), shift(1,1)], N).
  N = 2.


Extended Example: Rule Discovery and Application

The complete pattern induction workflow using ruleinfer + sceneapply:

  solve_task(TrainingPairs, TestInput, Output) :-
      % Infer from first pair
      TrainingPairs = [Before-After | _],
      % Try single-step rules
      (   rule_inference_infer_recolor(Before, After, Old, New),
          Rule = recolor(Old, New)
      ;   rule_inference_infer_shift(Before, After, DR, DC),
          Rule = shift(DR, DC)
      ;   rule_inference_infer_keep_color(Before, After, Color),
          Rule = keep_color(Color)
      ;   rule_inference_infer_remove_color(Before, After, Color),
          Rule = remove_color(Color)
      ;   rule_inference_infer_to_origin(Before, After),
          Rule = to_origin
      ),
      % Verify rule across all training pairs
      scene_apply_verify_all(Rule, TrainingPairs),
      % Apply rule to test input
      scene_apply_apply(Rule, TestInput, Output).

Summary

The sceneapply pack completes the rule-handling triad alongside scenecmp
(detects change) and ruleinfer (identifies the rule). Once a rule is inferred
and verified, sceneapply executes it on any scene. Its rule term vocabulary
covers all transformation types provided by scenexf, making it the symbolic
engine for hypothesis-driven scene transformation.
