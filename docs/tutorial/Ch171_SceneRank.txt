Chapter 171: scenerank Pack - Rule Hypothesis Ranking for Scene Lists (rk_*, Layer 189)


Overview

The scenerank pack ranks candidate symbolic rule terms by how many Before-After
scene pair training examples they correctly explain. Given a list of pairs and
a list of candidate rule terms, it answers: which rule fits best?

A Before-After pair is expressed as the term Before-After, where Before and
After are obj(Color, Cells) scene lists. Coverage is measured by applying the
rule to Before and comparing the result to After using msort (order-insensitive).

Layer 189. Pack prefix: rk_*. Work Package 210.


Coverage

The fundamental concept is coverage: the number of pairs that a rule correctly
explains. A rule explains a pair if applying the rule to the Before scene
produces a result order-insensitively equal to the After scene.


Exported Predicates


rk_n_pairs(+Pairs, -N)

Number of pairs in the pair list.


rk_coverage_pairs(+Rule, +Pairs, -Covered)

List of pairs that the rule correctly explains.

Example:

Pairs = [ [obj(r,[r(0,0)])]-[obj(g,[r(0,0)])],
          [obj(r,[r(1,0)])]-[obj(g,[r(1,0)])] ],
rk_coverage_pairs(recolor(r,g), Pairs, Covered).
% Covered = [ ... both pairs ... ]


rk_coverage(+Rule, +Pairs, -N)

Count of pairs the rule correctly explains.

Example:

rk_coverage(recolor(r,g), Pairs, N).
% N = 2  (if both pairs are r->g)


rk_consistent(+Rule, +Pairs)

Succeed if the rule correctly transforms every pair.

Example:

rk_consistent(recolor(r,g), Pairs).   % true if all pairs are r->g


rk_inconsistent_pairs(+Rule, +Pairs, -Bad)

List of pairs the rule does NOT explain.

Example:

rk_inconsistent_pairs(recolor(r,g), [GoodPair, BadPair], Bad).
% Bad = [BadPair]


rk_rule_score(+Rule, +Pairs, -N)

Alias for rk_coverage.


rk_rank_rules(+Rules, +Pairs, -Ranked)

Sort Rules by coverage descending. Ties are broken by standard Prolog term
order, giving a stable deterministic ordering.

Example:

rk_rank_rules([recolor(r,b), recolor(r,g)], Pairs, Ranked).
% Ranked = [recolor(r,g), recolor(r,b)]  (assuming r->g has higher coverage)


rk_rank_candidates(+Rules, +Pairs, -CovRulePairs)

Like rk_rank_rules but returns Coverage-Rule pairs so the coverage count is
available alongside the rule term.

Example:

rk_rank_candidates([recolor(r,g), recolor(r,b)], Pairs, CovRules).
% CovRules = [2-recolor(r,g), 0-recolor(r,b)]


rk_best_rule(+Rules, +Pairs, -Best)

The rule with the highest coverage. Fails if Rules is empty.

Example:

rk_best_rule([recolor(r,b), recolor(r,g)], Pairs, Best).
% Best = recolor(r,g)


rk_all_consistent(+Rules, +Pairs, -Consistent)

The subset of Rules that are fully consistent with every pair.

Example:

rk_all_consistent([recolor(r,g), recolor(r,b)], Pairs, Consistent).
% Consistent = [recolor(r,g)]  (if only that rule covers all pairs)


rk_filter_min_coverage(+Rules, +Pairs, +MinN, -Filtered)

Keep only rules whose coverage is >= MinN.

Example:

rk_filter_min_coverage([recolor(r,g), recolor(r,b)], Pairs, 2, Filtered).
% Filtered = [recolor(r,g)]  (if recolor(r,g) has coverage 2 >= 2)


rk_perfect_rules(+Rules, +Pairs, -Perfect)

Rules whose coverage equals the total pair count (perfect coverage).

Example:

rk_perfect_rules([recolor(r,g), recolor(r,b)], Pairs, Perfect).
% Perfect = [recolor(r,g)]  (if recolor(r,g) covers all pairs)


rk_coverage_ratio(+Rule, +Pairs, -Num/Den)

Coverage as an integer fraction. Num is the coverage count; Den is the
total pair count.

Example:

rk_coverage_ratio(recolor(r,g), Pairs, Num/Den).
% Num/Den = 2/3  (if recolor(r,g) covers 2 of 3 pairs)


rk_top_n_rules(+Rules, +Pairs, +N, -TopN)

The top N rules as Coverage-Rule pairs in descending coverage order.
If fewer than N candidates exist, returns all.

Example:

rk_top_n_rules([recolor(r,g), shift(1,0), recolor(r,b)], Pairs, 2, TopN).
% TopN = [2-recolor(r,g), 1-shift(1,0)]


Internal Rule Engine

scenerank implements its own rule application engine (rk_apply_) with the same
dispatch vocabulary as sceneapply: recolor(Old,New), recolor_all(New),
color_map(Map), shift(DR,DC), to_origin, reflect_h(Width), reflect_v(Height),
remove_color(Color), keep_color(Color), sort_size_desc, sort_size_asc, top_n(N).

This makes scenerank self-contained without importing sceneapply.


Typical Workflow

Given training pairs and a set of candidate rules generated by ruleinfer or
manual enumeration, use scenerank to select the best rule:

solve(TrainingPairs, TestInput, Output) :-
    Candidates = [recolor(r,g), recolor(b,r), shift(1,0), shift(-1,0),
                  to_origin, recolor_all(g)],
    rk_best_rule(Candidates, TrainingPairs, BestRule),
    rk_apply_(BestRule, TestInput, Output).


Dependencies

None. Uses library(lists) and library(apply).

Summary

The scenerank pack provides the selection step of the symbolic learning loop:
given a set of candidate rules, find the one that best explains the training data.
Combined with ruleinfer (inference) and sceneapply (execution), it completes
the learn-rank-apply pipeline for scene-level symbolic reasoning.
