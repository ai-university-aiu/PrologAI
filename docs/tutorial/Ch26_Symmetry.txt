Chapter 26 - Symmetry Pack
Layer 44 - Grid symmetry testing, canonical orientation, and orbit generation

PREREQUISITES

Chapter 6 (Grid Pack). The symmetry pack depends only on the grid pack.

Load:
  :- use_module(library(grid)).
  :- use_module(library(symmetry)).


WHAT IS THE SYMMETRY PACK?

The symmetry pack (module name: symmetry, file: packs/symmetry/prolog/symmetry.pl)
provides 12 sy_* predicates for testing whether a grid has various symmetry
properties, computing the full orbit of a grid under rotations and reflections,
finding canonical forms, and measuring symmetry order.

Symmetry in this context means: if you apply a transformation to the grid, the
result is equal to the original. The transformations tested come from the dihedral
group D4 (the symmetry group of the square) - which has 8 elements.


THE EIGHT D4 TRANSFORMS

The dihedral group D4 has 8 elements. The grid pack provides them all:

  gd_rotate90(Grid, Grid2)       - 90-degree clockwise rotation
  gd_rotate180(Grid, Grid2)      - 180-degree rotation
  gd_rotate270(Grid, Grid2)      - 270-degree clockwise rotation
  gd_reflect_h(Grid, Grid2)      - flip upside-down (top-bottom)
  gd_reflect_v(Grid, Grid2)      - flip left-right (horizontal)
  gd_reflect_d1(Grid, Grid2)     - transpose (main diagonal)
  gd_reflect_d2(Grid, Grid2)     - anti-diagonal flip
  (identity)                     - no change

The symmetry pack uses all eight to analyze grids.


INDIVIDUAL SYMMETRY TESTS

symmetry_is_hsymmetric(+Grid)
  Grid equals its left-right mirror (gd_reflect_v).

  ?- symmetry_is_hsymmetric([[1,2,1],[3,4,3]]).
  true.

symmetry_is_vsymmetric(+Grid)
  Grid equals its top-bottom mirror (gd_reflect_h).

  ?- symmetry_is_vsymmetric([[1,2,3],[4,5,6],[1,2,3]]).
  true.

symmetry_is_rot180(+Grid)
  Grid equals its 180-degree rotation.

  ?- symmetry_is_rot180([[1,2,3],[4,5,4],[3,2,1]]).
  true.

symmetry_is_rot90(+Grid)
  Grid equals its 90-degree clockwise rotation. Requires a square grid.

symmetry_is_diagonal(+Grid)
  Grid equals its main-diagonal reflection (transpose). Requires square.

symmetry_is_antidiagonal(+Grid)
  Grid equals its anti-diagonal reflection. Requires square.

All six tests fail for grids that lack the corresponding symmetry.


THE SYMMETRY GROUP

symmetry_group(+Grid, -Group)

Group is the list of symmetry atoms that hold for Grid. Atoms:
  h (horizontal/left-right), v (vertical/top-bottom), rot180, rot90,
  diag (main diagonal), antidiag (anti-diagonal).

  ?- symmetry_group([[1,2,1],[3,4,3]], Group).
  Group = [h].

  ?- symmetry_group([[1,1,1],[1,2,1],[1,1,1]], Group).
  Group = [h, v, rot180, rot90, diag, antidiag].

  ?- symmetry_group([[1,2,3],[4,5,6],[7,8,9]], Group).
  Group = [].


ORBITS AND ROTATIONS

symmetry_rotations(+Grid, -Rotations)

The distinct grids reachable by 0, 90, 180, and 270 degree rotations.
Duplicates are removed.

  ?- symmetry_rotations([[1,2],[3,4]], Rots),
     length(Rots, N).
  N = 4.  % All 4 rotations are distinct for an asymmetric grid.

  ?- symmetry_rotations([[1,1],[1,1]], Rots),
     length(Rots, N).
  N = 1.  % Uniform grid: all rotations are the same.

symmetry_orbit(+Grid, -Orbit)

The full dihedral orbit: all distinct grids reachable by any rotation or
reflection. Maximum 8 elements.

  ?- symmetry_orbit([[1,2,3],[4,5,6],[7,8,9]], Orbit),
     length(Orbit, N).
  N = 8.  % Asymmetric: full orbit of 8.

  ?- symmetry_orbit([[1,1,1],[1,2,1],[1,1,1]], Orbit),
     length(Orbit, N).
  N = 1.  % Fully symmetric: orbit is just the grid itself.


CANONICAL FORMS

symmetry_canonical(+Grid, -Canon)

The lexicographically smallest element of the orbit. Two grids have the
same canonical form if and only if they are equivalent under D4.

  ?- symmetry_canonical([[1,2,3],[4,5,6],[7,8,9]], C1),
     gd_rotate90([[1,2,3],[4,5,6],[7,8,9]], R90),
     symmetry_canonical(R90, C2),
     C1 = C2.
  true.  % Same canonical form: they are equivalent.


EQUIVALENCE

symmetry_equivalent(+GridA, +GridB)

Succeed if GridA and GridB are in the same dihedral orbit (same canonical form).

  ?- gd_rotate90([[1,2,3],[4,5,6],[7,8,9]], R90),
     symmetry_equivalent([[1,2,3],[4,5,6],[7,8,9]], R90).
  true.

  ?- symmetry_equivalent([[1,2,3],[4,5,6],[7,8,9]],
                   [[1,1,1],[1,2,1],[1,1,1]]).
  false.  % Different orbits.


SYMMETRY ORDER

symmetry_order(+Grid, -N)

N is the number of D4 transforms that map Grid to itself.
Computed as 8 divided by the orbit size.

  N = 1: fully asymmetric (orbit = 8)
  N = 2: one symmetry (e.g., rot180 only; orbit = 4)
  N = 4: two symmetries (orbit = 2)
  N = 8: all 8 symmetries (orbit = 1)

  ?- symmetry_order([[1,1,1],[1,2,1],[1,1,1]], N).
  N = 8.

  ?- symmetry_order([[1,2,3],[4,5,6],[7,8,9]], N).
  N = 1.


PREDICATE SUMMARY

Individual symmetry tests:
  symmetry_is_hsymmetric/1    - left-right symmetry
  symmetry_is_vsymmetric/1    - top-bottom symmetry
  symmetry_is_rot180/1        - 180-degree rotational symmetry
  symmetry_is_rot90/1         - 90-degree rotational symmetry (square)
  symmetry_is_diagonal/1      - main-diagonal symmetry (square)
  symmetry_is_antidiagonal/1  - anti-diagonal symmetry (square)

Group:
  symmetry_group/2            - all symmetry atoms that hold

Orbit:
  symmetry_rotations/2        - distinct rotations
  symmetry_orbit/2            - full dihedral orbit

Canonical form:
  symmetry_canonical/2        - lex-min element of orbit
  symmetry_equivalent/2       - orbit equivalence test
  symmetry_order/2            - symmetry order
