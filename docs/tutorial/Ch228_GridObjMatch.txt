Chapter 228: gridobjmatch - Object Matching and Change Detection (gom_*, Layer 246)

Work Package 267. Module: gridobjmatch. Prefix: gom_. Layer: 246.
Dependencies: library(lists) [member/2, subtract/3], library(apply) [maplist/2-3, include/3].


Overview

The gridobjmatch pack provides fourteen predicates for matching ob(Color,Cells,BBox)
object terms between two lists and detecting what changed. It answers the question:
"Given the objects in the input grid and the objects in the output grid, what moved,
what changed color, what appeared, and what disappeared?"

Objects have the form ob(Color, Cells, BBox) where BBox = r0(R0,C0,R1,C1).
Objects are produced by gridobj_all_objects/3 from the gridobj pack.

Match pairs use the term match(O1, O2).
Movement vectors use mv(DR, DC) where positive DR is down and positive DC is right.
Color change entries use cm(OldColor, NewColor).


1. Loading the Pack

:- use_module(packs/gridobjmatch/prolog/gridobjmatch).


2. Matching Objects Between Two Lists

Three strategies are available for pairing objects.

gridobjmatch_match_color(+Objs1, +Objs2, -Pairs)
  For each object in Objs1, find the first object in Objs2 with the same color.
  Matched objects in Objs2 are consumed (not reused). Unmatched objects are dropped.

?- Objs1 = [ob(r,[r(0,0)],r0(0,0,0,0)), ob(b,[r(1,1)],r0(1,1,1,1))],
   Objs2 = [ob(r,[r(0,2)],r0(0,2,0,2)), ob(b,[r(1,3)],r0(1,3,1,3))],
   gridobjmatch_match_color(Objs1, Objs2, Pairs).
% Pairs = [match(ob(r,...),ob(r,...)), match(ob(b,...),ob(b,...))]

gridobjmatch_match_nearest(+Objs1, +Objs2, -Pairs)
  For each object in Objs1, find the nearest object in Objs2 by doubled-centroid
  Manhattan distance. Nearest objects are consumed greedily.
  When Objs2 is exhausted, remaining Objs1 objects are unmatched.

?- O1 = ob(r,[r(0,0)],r0(0,0,0,0)),
   Near = ob(b,[r(0,1)],r0(0,1,0,1)),
   Far  = ob(g,[r(5,5)],r0(5,5,5,5)),
   gridobjmatch_match_nearest([O1], [Far, Near], Pairs).
% Pairs = [match(O1, Near)]   % selects nearest, not first

gridobjmatch_match_size(+Objs1, +Objs2, -Pairs)
  For each object in Objs1, find the first object in Objs2 with the same number of
  cells (cell count). Unmatched objects are dropped.


3. Finding Unmatched Objects

gridobjmatch_unmatched_a(+Pairs, +Objs1, -Unmatched)
  Unmatched is the sub-list of Objs1 with no partner in any match/2 pair.

gridobjmatch_unmatched_b(+Pairs, +Objs2, -Unmatched)
  Unmatched is the sub-list of Objs2 with no partner in any match/2 pair.

?- Pairs = [match(ob(r,...), ob(r,...))],
   Objs1 = [ob(r,...), ob(b,...), ob(g,...)],
   gridobjmatch_unmatched_a(Pairs, Objs1, UA).
% UA = [ob(b,...), ob(g,...)]


4. Detecting Color Changes

gridobjmatch_color_diff(+Pairs, -Same, -Diff)
  Partitions a match list into Same (both objects share the same color) and Diff
  (the two objects have different colors).

?- R1 = ob(r,[r(0,0)],r0(0,0,0,0)), R2 = ob(r,[r(0,1)],r0(0,1,0,1)),
   B1 = ob(b,[r(1,1)],r0(1,1,1,1)), G2 = ob(g,[r(1,1)],r0(1,1,1,1)),
   gridobjmatch_color_diff([match(R1,R2), match(B1,G2)], Same, Diff).
% Same = [match(R1,R2)], Diff = [match(B1,G2)]


5. Movement Vectors

gridobjmatch_move_vector(+match(O1,O2), -mv(DR,DC))
  DR and DC are the row and column displacement from the top-left bounding-box corner
  of O1 to that of O2. Positive DR = downward, positive DC = rightward.

gridobjmatch_move_vectors(+Pairs, -Vectors)
  Returns the list of mv/2 vectors for every pair.

gridobjmatch_constant_move(+Pairs, -mv(DR,DC))
  Succeeds if every pair has the same move vector. Fails for an empty list.

?- O1 = ob(r,[r(0,0)],r0(0,0,0,0)), O2 = ob(r,[r(0,2)],r0(0,2,0,2)),
   gridobjmatch_move_vector(match(O1,O2), V).
% V = mv(0, 2)

?- Pairs = [match(ob(r,_,r0(0,0,0,0)), ob(r,_,r0(0,1,0,1))),
            match(ob(b,_,r0(1,1,1,1)), ob(b,_,r0(1,2,1,2)))],
   gridobjmatch_constant_move(Pairs, V).
% V = mv(0, 1)


6. Inferring Color Maps

gridobjmatch_infer_color_map(+Pairs, -Map)
  Map is a sorted, deduplicated list of cm(OldColor,NewColor) for matched pairs
  where the two objects have different colors.

?- Pairs = [match(ob(r,_,_), ob(b,_,_)), match(ob(g,_,_), ob(y,_,_))],
   gridobjmatch_infer_color_map(Pairs, Map).
% Map = [cm(g,y), cm(r,b)]


7. Appeared and Disappeared Objects

gridobjmatch_appeared(+Objs1, +Objs2, -Appeared)
  Appeared contains objects in Objs2 whose color does not appear in any object
  in Objs1 (brand-new color classes).

gridobjmatch_disappeared(+Objs1, +Objs2, -Disappeared)
  Disappeared contains objects in Objs1 whose color does not appear in any object
  in Objs2 (lost color classes).

These predicates detect color-class changes, not individual object identity.


8. Structural Comparison

gridobjmatch_same_structure(+Objs1, +Objs2)
  Succeeds if both lists contain the same multiset of object colors. Order and
  position do not matter.

gridobjmatch_count_change(+Objs1, +Objs2, -N)
  N = length(Objs2) - length(Objs1). Positive = objects added; negative = removed;
  zero = same count.


9. Integration Example

The following shows a complete workflow for analyzing a transformation between
two object lists (typical in grid puzzle analysis).

:- use_module(packs/gridobj/prolog/gridobj).
:- use_module(packs/gridobjmatch/prolog/gridobjmatch).

analyze_transformation(GridIn, GridOut, BgColor, Analysis) :-
    gridobj_all_objects(GridIn,  BgColor, ObjsIn),
    gridobj_all_objects(GridOut, BgColor, ObjsOut),
    gridobjmatch_match_color(ObjsIn, ObjsOut, Pairs),
    gridobjmatch_color_diff(Pairs, Same, Diff),
    gridobjmatch_constant_move(Pairs, MoveVec),
    gridobjmatch_infer_color_map(Diff, ColorMap),
    gridobjmatch_appeared(ObjsIn, ObjsOut, New),
    gridobjmatch_disappeared(ObjsIn, ObjsOut, Lost),
    Analysis = analysis(move(MoveVec), color_map(ColorMap),
                        appeared(New), disappeared(Lost),
                        same_color(Same)).


10. Design Notes

The doubled-centroid technique in gridobjmatch_match_nearest uses R0+R1 and C0+C1 instead
of dividing by 2. This produces exact integer arithmetic with no rounding error.
The centroid ordering is preserved: the nearest object in Euclidean space is also
the nearest by doubled-centroid Manhattan distance.

gridobjmatch_constant_move intentionally fails for an empty list. The absence of any pairs
provides no evidence of a constant move; the predicate should not succeed vacuously.

gridobjmatch_appeared and gridobjmatch_disappeared work on color classes. If red appears in both
lists but with different cell sets, these predicates are silent. They detect
structural change at the level of "which colors are present," not "where are they."

Private helpers use the trailing-underscore convention: gridobjmatch_color_/2, gridobjmatch_size_/2,
gridobjmatch_top_left_/3, gridobjmatch_centroid_/3, gridobjmatch_select_color_/4, gridobjmatch_select_size_/4,
gridobjmatch_find_nearest_/3, gridobjmatch_remove_once_/3, gridobjmatch_has_vector_/2, gridobjmatch_new_color_/2,
gridobjmatch_gone_color_/2.
