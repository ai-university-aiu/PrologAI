pocket_shot solves ARC-AGI-2 task 8b9c3697. The input grid contains
frame-shaped structures made of an arm color (the unique non-background
non-2 color), plus one or more rectangular blocks of 2-cells. Each
2-block either fires into the interior of a frame structure (leaving a
trail of 0s and landing adjacent to an interior peg) or disappears
(becomes background).


Algorithm:

  Phase 1 - Detect background:
  1. The background color BG is the most common cell value (via arc2_bg_color_).

  Phase 2 - Find 2-blocks:
  2. Collect all cells holding value 2 using findall.
  3. Group them into 4-connected components using arc2_bfs_/7.
  4. Compute the axis-aligned bounding box (R1-R2-C1-C2) for each component.

  Phase 3 - Determine block fate:
  5. For each block, try all four firing directions: up, down, left, right.
  6. For vertical firing (up/down), scan each column of the block from the
     leading edge (top row for up, bottom row for down) in that direction,
     skipping BG and 2-cells, to find the first arm-cell.
     For horizontal firing (left/right), scan each row from the leading edge.
  7. Block integrity check: all arm-cells found for the block must be at the
     same row (vertical) or same column (horizontal). This ensures the block
     moves as a rigid unit. If arm-cells are at different positions, this
     direction fails.
  8. Interior peg validation: each arm-cell (XR, XC) must satisfy three
     conditions:
     Condition 1 - Open face: the cell in the opposite direction from (XR, XC)
       is BG or off-grid (the arm-cell faces the 2-block with an open side).
     Condition 2 - Parent arm: the cell one step further in the firing direction
       from (XR, XC) is an arm-cell (the arm continues deeper into the shape).
     Condition 3 - Perpendicular bar: the parent cell has at least 2 arm-cell
       neighbors in the two perpendicular directions (the parent is part of a
       bar or spine, not just a tip or corner).
  9. If all block positions find valid pegs at the same distance, the direction
     is valid for this block. A block fires in the unique valid direction.

  Phase 4 - Apply fate:
  10. Firing block: the original block positions and all trail cells (the path
      between the landing position and the original, exclusive of landing)
      become 0. The landing position (one step from the arm-cells in the
      direction opposite to firing) becomes 2.
  11. Non-firing block (no valid direction): all block cells become BG.

  Phase 5 - Build output:
  12. Priority: landing cells -> 2; trail/original cells -> 0; erased cells ->
      BG; all other cells copy from input.

Key predicates:

arc2_ps_scan_(+Grid, +BG, +R, +C, +D, -XR-XC):
  Starts one step from (R,C) in direction D. Skips BG and 2-cells via
  arc2_ps_scan_go_/7. Returns first arm-cell (XR,XC). Fails at grid boundary.

arc2_ps_valid_peg_(+Grid, +BG, +XR-XC, +D):
  Three-condition check. Uses arc2_ps_opp_/2 for opposite direction,
  arc2_ps_perpdirs_/2 for perpendicular list, and include/3 with
  arc2_ps_arm_nbr_ to count perpendicular arm-neighbors of the parent.

arc2_ps_peg_col_(+Grid, +BG, +SR, +D, +C, -XR):
  Scans column C from row SR in direction D; returns arm row XR if valid peg.
  Used by maplist in arc2_ps_fire_ for vertical firing.

arc2_ps_fire_(+Grid, +BG, +Box, -D, -Land):
  Tries all four directions via member/2. maplist(arc2_ps_peg_col_) or
  maplist(arc2_ps_peg_row_) collects arm positions. sort(XRs,[XAR]) enforces
  block integrity. vland_/hland_ computes landing box.

arc2_ps_comps_(+Grid, +Twos, +Seen, -Boxes):
  Groups 2-cells into bounding boxes using arc2_bfs_/7 for connectivity and
  min_list/max_list for bbox. Skips already-seen cells.

arc2_ps_trail_(+Box, +D, +Land, -Zeros, -Twos):
  Four direction clauses. Uses findall with between/3 to enumerate zero-cells
  and two-cells.

arc2_ps_solve_acc_/9:
  Accumulates zero/two/erase sets across all blocks. once(arc2_ps_fire_)
  determines fate; appends results; no backtracking in apply phase.


Key insights:

Interior peg identification: condition 3 (perpendicular bar test) is the
critical filter. A cell at the end of a thin arm (tip) passes conditions 1-2
but fails condition 3 because its parent has no perpendicular neighbors. Only
cells whose parent is part of a bar or spine (flanked on both sides by arm
cells) are valid landing targets.

Block integrity: requiring all arm-cells to be at the same row (or column)
ensures the block moves as a rigid unit. This naturally handles rectangular
blocks of any size: a 2x3 block needs all 3 arm-cells (one per column) to be
at the same row for valid firing.

Fast induction: a specific arc2_induce_rule clause with a 2-cell pre-filter
is placed before the generic catch-all. This prevents timeout on large grids
where slow rules (e.g., jigsaw_fill) would otherwise exhaust the 10-second
per-task time limit before pocket_shot is reached.
