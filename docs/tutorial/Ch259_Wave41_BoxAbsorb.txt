Chapter 259: Wave 41 - box_absorb (WP-299, Layer 274)

WHAT IT DOES

box_absorb solves ARC-AGI-2 task dd6b8c4b. Every input is an 11x11 grid with
background value 7. A 3x3 box of 3s (with the value 2 at its centre) sits
somewhere in the grid, enclosed fully or partially by a 6-frame. Scattered 9s
appear throughout the grid. The rule uses a BFS flood from the box outward
through all non-6 cells to discover 9s in order of their distance from the box.
It absorbs up to 9 of them (the box capacity), removes them from their original
positions, and fills the box from the top-left in reading order.

THE BOX

arc2_bxa_find2_ locates the unique 2 cell by scanning rows top to bottom with
nth0 and cuts on the first match. The 3x3 box is the nine cells centred on the
2: rows CR-1 to CR+1, cols CC-1 to CC+1.

BFS ABSORPTION

arc2_bxa_bfs_ runs a strict FIFO BFS (new cells appended to the back of the
queue) starting from all nine box cells simultaneously. The BFS expands through
all non-6 cells (background 7s, absorbed 9s, and box interior cells). When a
9-cell is encountered as a new neighbour, it is absorbed and added to the queue
so the BFS can continue through it. Absorption halts when the accumulated count
reaches 9 (the box capacity) or the queue empties.

arc2_bxa_expand_ uses findall with cardinal-neighbour offsets to collect all
unvisited non-6 neighbours (New). include with arc2_bxa_is9_ then filters New
down to the subset whose grid value is 9 (Nine). Both lists are returned so the
caller can enqueue all new cells and track which are absorbed.

FILLING THE BOX

After BFS, the absorbed count N is used to take the first N cells of BoxCells
(which is already in reading order from numlist + findall). These N cells become
the Slots that receive value 9 in the output. Absorbed 9s outside the box
become 7. All other cells keep their original values.

arc2_bxa_build_ uses a double maplist over row and column indices to build the
output grid cell by cell using three-way conditional dispatch.

EXAMPLE: PAIR 3 (simplest)

Input has no 6-frame at all. Four 9s at (1,9),(2,2),(2,10),(9,8) are scattered
around the 3x3 box at rows 4-6, cols 4-6. BFS immediately discovers all four
9s (none blocked by 6-cells). N=4, box not fully capped. Output fills box cells
(4,4),(4,5),(4,6),(5,4) with 9 and clears the four original 9 positions.

EXAMPLE: PAIR 2 (cap reached)

Input has a U-shaped 6-frame (row3 top bar, col3/col7 side walls, open bottom).
Fifteen 9s are in the grid. BFS expands downward through the open bottom into
rows 7-10 and around the frame via col0. It absorbs 9 cells in BFS distance
order before capping. The six 9s at rows 0-2 (farther away) remain in place.

PREDICATES

  arc2_transform(box_absorb, +Grid, -Out)
      Entry point. Calls find2_, bfs_, then build_ in sequence.

  arc2_bxa_find2_(+Grid, -R, -C)
      nth0 row scan with cut; returns row and column of the 2 cell.

  arc2_bxa_bfs_(+Grid, +Queue, +Visited, +Cap, +Acc, -Absorbed)
      FIFO BFS loop with capacity counter; halts at Cap=0 or empty queue.

  arc2_bxa_expand_(+RC, +Grid, +Visited, -New, -Nine)
      findall over cardinal offsets; include filters for 9-valued cells.

  arc2_bxa_is9_(+Grid, +RC)
      nth0 double-lookup; succeeds iff cell value is 9.

  arc2_bxa_build_(+Grid, +Absorbed, +Slots, -Out)
      Double maplist with three-way conditional for box/absorbed/original.

KEY DESIGN NOTES

The FIFO BFS order is essential. A DFS expansion discovers cells in the wrong
order, absorbing nearby upper-left 9s before reaching more distant but
frame-adjacent ones. Only BFS (cells at equal distance before cells farther
away) reproduces the expected output for all pairs.

The cap at 9 (box capacity) acts as the stopping condition when the accessible
9s outnumber the box slots. Once 9 cells are absorbed, the BFS halts even if
more 9s are reachable; those extra 9s remain unchanged in the output.
