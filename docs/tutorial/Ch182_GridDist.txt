Chapter 182: griddist Pack - Grid Distance Transform (gd_*, Layer 200)


Overview

The griddist pack computes spatial distances between cells in raw grid format
(list-of-lists of color atoms). It provides point-to-point distance formulas,
per-cell distance maps to the nearest instance of a color, BFS-based flood
distances with obstacles, and higher-level operations built on those maps:
zone marking, ring extraction, Voronoi assignment, N-step morphological
expand and shrink, distance-based recoloring, and equidistance detection.

Layer 200. Pack prefix: gd_*. Work Package 221.


Exported Predicates


griddist_manhattan(+R1, +C1, +R2, +C2, -D)

D is the Manhattan (L1, city-block) distance between grid positions (R1,C1)
and (R2,C2): the sum of the absolute row difference and absolute column
difference.

Example:

griddist_manhattan(0, 0, 1, 1, D).  % D = 2
griddist_manhattan(0, 0, 0, 3, D).  % D = 3


griddist_chebyshev(+R1, +C1, +R2, +C2, -D)

D is the Chebyshev (L-infinity, chessboard) distance: the maximum of the
absolute row difference and absolute column difference. A diagonal step costs
the same as a cardinal step.

Example:

griddist_chebyshev(0, 0, 1, 1, D).  % D = 1 (not 2 as in Manhattan)
griddist_chebyshev(0, 0, 2, 3, D).  % D = 3


griddist_dist_map(+Grid, +Color, -DistGrid)

DistGrid is a raw grid of the same dimensions where each cell holds the
minimum Manhattan distance from that cell to the nearest cell in Grid whose
value equals Color. If Color is absent from Grid, every cell gets 9999.

Example:

% b at center (1,1) of 3x3 grid
griddist_dist_map([[r,r,r],[r,b,r],[r,r,r]], b, DMap).
% DMap = [[2,1,2],[1,0,1],[2,1,2]]


griddist_dist_map8(+Grid, +Color, -DistGrid)

Like griddist_dist_map but uses Chebyshev distance. Diagonal neighbors count as
distance 1, so corners in the example above become 1 rather than 2.

Example:

griddist_dist_map8([[r,r,r],[r,b,r],[r,r,r]], b, DMap8).
% DMap8 = [[1,1,1],[1,0,1],[1,1,1]]


griddist_nearest_coord(+Grid, +R, +C, +Color, -Coord)

Coord is [R2,C2] where (R2,C2) is the nearest cell to (R,C) whose value
equals Color. Ties are broken by standard term order (top-left wins).
Fails if no Color cell exists in Grid.

Example:

griddist_nearest_coord([[r,r,r],[r,b,r],[r,r,r]], 0, 0, b, Coord).
% Coord = [1,1]


griddist_zone(+Grid, +Color, +N, -ZoneGrid)

ZoneGrid is Grid with every non-Color cell within Manhattan distance N of any
Color cell replaced by the atom zone. Color cells keep their original value.
Cells farther than N from all Color cells keep their original value.

Example:

griddist_zone([[r,r,r],[r,b,r],[r,r,r]], b, 1, ZG).
% ZG = [[r,zone,r],[zone,b,zone],[r,zone,r]]


griddist_ring(+Grid, +R0, +C0, +D, -Coords)

Coords is the list of [R,C] pairs within Grid at exactly Manhattan distance D
from (R0,C0). Out-of-bounds positions are excluded. D=0 returns [[R0,C0]].

Example:

griddist_ring([[r,r,r],[r,b,r],[r,r,r]], 1, 1, 1, Ring).
% Ring = [[0,1],[1,0],[1,2],[2,1]]

griddist_ring([[r,r,r],[r,b,r],[r,r,r]], 1, 1, 2, Ring2).
% Ring2 = [[0,0],[0,2],[2,0],[2,2]]


griddist_voronoi(+Grid, -VGrid)

VGrid is Grid with every background cell replaced by the color of the nearest
non-background cell (Manhattan distance). Background is detected as the most
frequent color. Ties broken by standard term order. If no foreground cells
exist, VGrid equals Grid unchanged.

Example:

% b at (1,0), g at (1,2), r background
griddist_voronoi([[r,r,r],[b,r,g],[r,r,r]], VG).
% VG = [[b,b,g],[b,b,g],[b,b,g]]  (equidistant cells get b since b < g)


griddist_bfs_flood(+Grid, +R0, +C0, +Wall, -DistGrid)

DistGrid is a raw grid of BFS distances from (R0,C0) treating cells with
value Wall as impassable. The start cell gets distance 0. Unreachable cells
and Wall cells get 9999. Fails if the start cell itself has value Wall.

Example:

% b at center acts as wall; path to (1,2) detours via (0,2)
griddist_bfs_flood([[r,r,r],[r,b,r],[r,r,r]], 0, 0, b, DG).
% DG[1][2] = 3  (path: 0,0 -> 0,1 -> 0,2 -> 1,2)
% DG[1][1] = 9999  (blocked wall)


griddist_recolor_by_dist(+Grid, +Color, +Palette, -NewGrid)

Each Color cell at Manhattan distance D from the nearest background cell is
recolored to the D-th element of Palette (1-indexed). Color cells with D
greater than the palette length keep their original Color value. Non-Color
cells are unchanged. Background is detected as the most frequent color.

Example:

% 5x5 grid with 3x3 b block; center b at dist 2, border b at dist 1
griddist_recolor_by_dist(G5x5, b, [red, blue], NewG).
% Border b cells -> red; center b -> blue


griddist_expand_n(+Grid, +Color, +N, -Expanded)

Expanded is Grid with every cell within Manhattan distance N of any Color
cell set to Color. This is N-step morphological dilation of Color regions.
N=0 leaves the grid unchanged.

Example:

% Expand single center b outward by 1 step
griddist_expand_n([[r,r,r],[r,b,r],[r,r,r]], b, 1, E).
% E = [[r,b,r],[b,b,b],[r,b,r]]


griddist_shrink_n(+Grid, +Color, +N, -Shrunk)

Shrunk is Grid with every Color cell within Manhattan distance N of the
nearest background cell replaced by background. This is N-step morphological
erosion of Color regions. N=0 leaves the grid unchanged.

Example:

% 5x5 grid 3x3 b block: border b cells at dist 1 eroded; center b at dist 2 survives
griddist_shrink_n(G5x5, b, 1, S).
% S has b only at (2,2)


griddist_equidistant(+Grid, +ColorA, +ColorB, -EqGrid)

EqGrid is Grid with every cell where the Manhattan distance to ColorA equals
the Manhattan distance to ColorB replaced by the atom equidist. ColorA and
ColorB cells keep their original value even if they happen to be equidistant.

Example:

% b at (0,0), g at (2,2); cells on the anti-diagonal at equal distance
griddist_equidistant([[b,r,r],[r,r,r],[r,r,g]], b, g, EqG).
% EqG[1][1] = equidist, EqG[0][2] = equidist, EqG[2][0] = equidist


griddist_max_dist(+Grid, +Color, -R, -C)

(R,C) is the position of the cell farthest (by Manhattan distance) from any
Color cell. Ties resolved by row-major order (top-left). Fails if Grid
contains no Color cells.

Example:

% b at center; all corners are at dist 2; top-left corner wins
griddist_max_dist([[r,r,r],[r,b,r],[r,r,r]], b, R, C).
% R = 0, C = 0


Typical Use Cases

Proximity zone (mark cells near a region):
  griddist_zone(Grid, Color, 2, ZoneGrid)

Voronoi assignment (partition by nearest foreground):
  griddist_voronoi(Grid, VGrid)

Path planning around obstacles:
  griddist_bfs_flood(Grid, StartR, StartC, WallColor, DistGrid)

Morphological grow/shrink by N steps:
  griddist_expand_n(Grid, Color, 3, Expanded)
  griddist_shrink_n(Grid, Color, 2, Shrunk)

Distance-gradient recoloring:
  griddist_recolor_by_dist(Grid, Color, [outer, middle, inner], NewGrid)

Equidistant boundary detection:
  griddist_equidistant(Grid, ColorA, ColorB, EqGrid)


Dependencies

None. Uses library(lists) and built-in predicates only.

Summary

The griddist pack adds the distance-transform layer to PrologAI's raw-grid
toolkit. Combined with gridnbr (immediate neighbors, Layer 199), it provides
both local and global spatial reasoning: how far is each cell from a given
color, which cells are nearest to which foreground objects, and how regions
grow or shrink when dilated or eroded by N steps.
