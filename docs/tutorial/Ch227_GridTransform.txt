Chapter 227: gridtransform - Grid Transformation Detection and Application (gtr_*, Layer 245)

Work Package 266. Module: gridtransform. Prefix: gtr_. Layer: 245.
Dependencies: library(lists) [member/2].
Requirement: GridA and GridB must have the same dimensions for all predicates.


Overview

The gridtransform pack provides fourteen predicates for detecting color mappings
between two grids, computing cell-level differences, and applying transformations.

Color maps are represented as sorted lists of cm(OldColor, NewColor) terms.
Change lists are lists of chg(R, C, OldColor, NewColor) terms.


Predicates

gridtransform_color_map(+GridA, +GridB, -Map)
  Map is the consistent color mapping from GridA to GridB: a sorted list of
  cm(OldColor, NewColor) pairs. Fails if any source color maps to more than one
  destination at different positions (inconsistent mapping).

gridtransform_apply_color_map(+Grid, +Map, -Result)
  Result is Grid with each cell color replaced per Map. Unmapped colors pass through.

gridtransform_diff_cells(+GridA, +GridB, -Cells)
  Cells is the list of r(R,C) positions where the grids have different values.

gridtransform_diff_count(+GridA, +GridB, -N)
  N is the count of positions where the grids differ.

gridtransform_same_cells(+GridA, +GridB, -Cells)
  Cells is the list of r(R,C) positions where the grids agree.

gridtransform_changed_colors(+GridA, +GridB, -Changes)
  Changes is a list of chg(R,C,OldColor,NewColor) for each position that changed.

gridtransform_invert_map(+Map, -InvMap)
  InvMap swaps source and target: cm(VB,VA) for each cm(VA,VB) in Map.
  Fails if Map is not injective (two sources map to the same target).

gridtransform_compose_maps(+MapAtoB, +MapBtoC, -MapAtoC)
  MapAtoC maps colors from MapAtoB's domain through MapBtoC. Only colors present
  in both maps appear in MapAtoC.

gridtransform_is_identity_map(+Map)
  Succeed if every cm(VA,VB) in Map has VA = VB. Succeeds for empty map.

gridtransform_apply_changes(+Grid, +Changes, -Result)
  Result is Grid with each chg(R,C,_,NewColor) applied. For deterministic behavior,
  provide at most one change per cell position.

gridtransform_delta_grid(+GridA, +GridB, +Bg, -Delta)
  Delta shows only changed cells: changed positions use GridB's value; unchanged
  positions use Bg. Composes with gridtransform_overlay: overlay(A, delta(A,B,Bg), Bg) = B.

gridtransform_overlay(+BaseGrid, +DeltaGrid, +Bg, -Result)
  Result is BaseGrid with every non-Bg cell of DeltaGrid pasted on top.

gridtransform_common_grid(+GridA, +GridB, +Bg, -Common)
  Common keeps cells where both grids agree; differing cells become Bg.

gridtransform_is_color_permutation(+GridA, +GridB)
  Succeed if GridB is a bijective color permutation of GridA: consistent mapping
  that is also injective (no two source colors map to the same target).


Color map consistency and inversion

gridtransform_color_map builds the map by collecting all cm(VA,VB) pairs from corresponding
cell positions, deduplicating with sort, then checking that no source color VA
appears with two different targets. This means a valid map might be partial
(not every color of GridA need appear if the grid has no cells of that color).

gridtransform_invert_map swaps keys and values; it fails if the inverse would be
inconsistent (two different VA values mapped to the same VB in the original).

gridtransform_compose_maps produces A-to-C by chaining A-to-B and B-to-C. Colors from A
that have no matching intermediate color in B-to-C are silently dropped.


Example session

:- use_module(pack(gridtransform/prolog/gridtransform)).

% Detect color mapping.
A = [[r,b],[b,r]],
B = [[b,r],[r,b]],
gridtransform_color_map(A, B, Map).
% Map = [cm(b,r), cm(r,b)].

% Apply map to a grid.
gridtransform_apply_color_map([[r,r,b]], [cm(r,g), cm(b,r)], Result).
% Result = [[g,g,r]].

% Diff and delta.
gridtransform_diff_count([[r,b],[b,r]], [[r,b],[g,r]], 1).
gridtransform_delta_grid([[r,b],[b,r]], [[r,b],[g,r]], x, [[x,x],[g,x]]).
