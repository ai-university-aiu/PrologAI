Chapter 132: Arrange Pack

The arrange pack (ag_*, Layer 150) provides 14 predicates for reasoning about the
spatial arrangement of obj(Color, Cells) terms as produced by the scene pack. Where
the pair pack (Layer 149) answers "are these two objects the same?", the arrange pack
answers "where are these objects relative to each other?" and "how are they laid out?"

All predicates operate on obj(Color, Cells) terms where Cells is a sorted list of
r(R,C) terms. The central concept is the centroid: the integer (floor-average) row
and column of the cell list.


Object Centroid

arrange_centroid(+Obj, -r(CR,CC)): compute the floor-average centroid of an obj term.
The centroid row is the sum of all cell rows divided by the cell count (// operator).
The centroid column is computed the same way.

    arrange_centroid(obj(1, [r(4,6)]), C).
    % C = r(4,6)   (single cell; centroid is the cell itself)

    arrange_centroid(obj(2, [r(0,0), r(0,2)]), C).
    % C = r(0,1)   (sum of cols = 0+2=2; 2//2=1)

    arrange_centroid(obj(3, [r(0,0), r(0,1), r(1,0), r(1,1)]), C).
    % C = r(0,0)   (sum of rows = 0+0+1+1=2; 2//4=0)

arrange_offset(+Obj1, +Obj2, -r(DR,DC)): displacement vector from Obj1 centroid to Obj2.

    arrange_offset(obj(1,[r(0,0)]), obj(2,[r(0,3)]), Offset).
    % Offset = r(0,3)

    arrange_offset(obj(1,[r(1,1)]), obj(2,[r(3,4)]), Offset).
    % Offset = r(2,3)


Pairwise Spatial Order

arrange_row_order(+Obj1, +Obj2): Obj1 centroid row is at most Obj2 centroid row.

    arrange_row_order(obj(1,[r(0,0)]), obj(2,[r(3,0)])).  % succeeds (0 =< 3)
    arrange_row_order(obj(1,[r(2,0)]), obj(2,[r(2,5)])).  % succeeds (equal rows)
    % arrange_row_order(obj(1,[r(5,0)]), obj(2,[r(1,0)])).  % fails (5 > 1)

arrange_col_order(+Obj1, +Obj2): Obj1 centroid column is at most Obj2 centroid column.

    arrange_col_order(obj(1,[r(0,0)]), obj(2,[r(0,4)])).  % succeeds (0 =< 4)
    % arrange_col_order(obj(1,[r(0,7)]), obj(2,[r(0,2)])).  % fails (7 > 2)

arrange_row_aligned(+Obj1, +Obj2): both objects share the same centroid row.

    arrange_row_aligned(obj(1,[r(2,0)]), obj(2,[r(2,5)])).  % succeeds
    % arrange_row_aligned(obj(1,[r(0,0)]), obj(2,[r(1,0)])).  % fails

arrange_col_aligned(+Obj1, +Obj2): both objects share the same centroid column.

    arrange_col_aligned(obj(1,[r(0,3)]), obj(2,[r(4,3)])).  % succeeds


Sorting Objects by Position

arrange_sort_by_row(+Objs, -Sorted): sort obj list top to bottom by centroid row.
Uses keysort/2, which is stable: equal-row objects preserve their input order.

    Objs = [obj(1,[r(3,0)]), obj(2,[r(1,0)]), obj(3,[r(2,0)])],
    arrange_sort_by_row(Objs, Sorted).
    % Sorted = [obj(2,[r(1,0)]), obj(3,[r(2,0)]), obj(1,[r(3,0)])]

arrange_sort_by_col(+Objs, -Sorted): sort obj list left to right by centroid column.

    Objs = [obj(1,[r(0,5)]), obj(2,[r(0,1)]), obj(3,[r(0,3)])],
    arrange_sort_by_col(Objs, Sorted).
    % Sorted = [obj(2,[r(0,1)]), obj(3,[r(0,3)]), obj(1,[r(0,5)])]


Gap Analysis

arrange_row_gaps(+Objs, -Gaps): consecutive gaps between sorted distinct centroid rows.
Deduplicates centroid rows first (sort/2), so coincident objects count once.

    Objs = [obj(1,[r(0,0)]), obj(2,[r(3,0)]), obj(3,[r(7,0)])],
    arrange_row_gaps(Objs, Gaps).
    % Gaps = [3, 4]

    Objs2 = [obj(1,[r(0,0)]), obj(2,[r(2,0)]), obj(3,[r(4,0)])],
    arrange_row_gaps(Objs2, Gaps2).
    % Gaps2 = [2, 2]

arrange_col_gaps(+Objs, -Gaps): consecutive gaps between sorted distinct centroid columns.

    Objs = [obj(1,[r(0,0)]), obj(2,[r(0,3)]), obj(3,[r(0,6)])],
    arrange_col_gaps(Objs, Gaps).
    % Gaps = [3, 3]

arrange_equal_row_gaps(+Objs): all consecutive centroid row gaps are equal.
Requires at least two objects with distinct centroid rows.

    Objs = [obj(1,[r(0,0)]), obj(2,[r(3,0)]), obj(3,[r(6,0)])],
    arrange_equal_row_gaps(Objs).  % succeeds (both gaps are 3)

    Objs2 = [obj(1,[r(0,0)]), obj(2,[r(2,0)]), obj(3,[r(5,0)])],
    % arrange_equal_row_gaps(Objs2).  % fails (gaps are 2 and 3)

arrange_equal_col_gaps(+Objs): all consecutive centroid column gaps are equal.

    Objs = [obj(1,[r(0,0)]), obj(2,[r(0,4)]), obj(3,[r(0,8)])],
    arrange_equal_col_gaps(Objs).  % succeeds (both gaps are 4)


Group Bounding Box

arrange_group_bbox(+Objs, -R0, -C0, -R1, -C1): bounding box of all cells of all objs.
This is not the box around centroids but around the actual cells.

    Objs = [obj(1,[r(0,0), r(0,1)]), obj(2,[r(3,2), r(3,3)])],
    arrange_group_bbox(Objs, R0, C0, R1, C1).
    % R0=0, C0=0, R1=3, C1=3

    Objs2 = [obj(1,[r(0,0)]), obj(2,[r(5,0)]), obj(3,[r(0,8)])],
    arrange_group_bbox(Objs2, R0, C0, R1, C1).
    % R0=0, C0=0, R1=5, C1=8


Nearest Neighbor

arrange_nearest(+RefObj, +Objs, -Nearest): find the obj in Objs whose centroid is
closest (Manhattan distance) to RefObj's centroid.

    Ref = obj(1, [r(0,0)]),
    Candidates = [obj(2,[r(0,5)]), obj(3,[r(0,2)])],
    arrange_nearest(Ref, Candidates, Nearest).
    % Nearest = obj(3,[r(0,2)])   (distance 2 vs 5)

    Ref2 = obj(1, [r(5,5)]),
    Cands2 = [obj(2,[r(0,0)]), obj(3,[r(4,4)]), obj(4,[r(9,9)])],
    arrange_nearest(Ref2, Cands2, N).
    % N = obj(3,[r(4,4)])   (Manhattan distance 2)
