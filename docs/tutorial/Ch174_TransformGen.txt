Chapter 174: transformgen Pack - Systematic Generation of Scene Transformation Rule Candidates (tg_*, Layer 192)


Overview

The transformgen pack systematically generates sets of candidate rule terms
from scene data or training pairs. Where gridsolve uses heuristic inference
to guess the most likely rules, transformgen exhaustively enumerates every
plausible rule within a bounded search space.

Layer 192. Pack prefix: tg_*. Work Package 213.


Why Enumeration

Heuristic inference works well for common patterns but may miss unusual rules
or edge cases. By generating ALL recolor variants, ALL shift offsets within
bounds, ALL color maps, and ALL filter rules derivable from the scene data,
transformgen gives the filtering layer (tg_filter_consistent) a complete
search space. A rule cannot be missed if it was generated.


Exported Predicates


tg_scene_colors(+Scene, -Colors)

All distinct color atoms in the scene, sorted.

Example:

tg_scene_colors([obj(r,[r(0,0)]), obj(b,[r(0,1)])], Colors).
% Colors = [b, r]


tg_recolor_candidates(+Scene, -Candidates)

All recolor(Old,New) rules for distinct color pairs in the scene.

Example:

tg_recolor_candidates([obj(r,[r(0,0)]), obj(b,[r(0,1)])], Cands).
% Cands = [recolor(r,b), recolor(b,r)]


tg_recolor_all_candidates(+Scene, -Candidates)

All recolor_all(C) rules, one per color in the scene.


tg_remove_candidates(+Scene, -Candidates)

All remove_color(C) rules, one per color.


tg_keep_candidates(+Scene, -Candidates)

All keep_color(C) rules, one per color.


tg_shift_candidates(+MaxDR, +MaxDC, -Candidates)

All shift(DR,DC) rules with |DR| =< MaxDR and |DC| =< MaxDC, excluding (0,0).

Example:

tg_shift_candidates(1, 1, Cands).
% 8 candidates: all (DR,DC) combinations in [-1..1] x [-1..1] except (0,0)


tg_reflect_candidates(+GridH, +GridW, -Candidates)

reflect_h(GridW) and reflect_v(GridH) for the given grid dimensions.

Example:

tg_reflect_candidates(5, 7, Cands).
% Cands = [reflect_h(7), reflect_v(5)]


tg_top_candidates(+MaxN, +Scene, -Candidates)

All top_n(K) rules for K in 1..min(MaxN, |Scene|).

Example:

tg_top_candidates(3, [obj(r,[r(0,0)]), obj(b,[r(0,1)])], Cands).
% Cands = [top_n(1), top_n(2)]  (capped at 2 since scene has 2 objects)


tg_color_map_candidate(+Before, +After, -Rule)

Derive a color_map rule from one Before-After pair (positional matching).
Fails if no colors change.

Example:

Before = [obj(r,[r(0,0)]), obj(b,[r(0,1)])],
After  = [obj(g,[r(0,0)]), obj(y,[r(0,1)])],
tg_color_map_candidate(Before, After, Rule).
% Rule = color_map([r-g, b-y])


tg_from_scenes(+Before, +After, -Candidates)

Generate all rule candidates from one Before-After pair. Union of: recolor
from Before colors, recolor_all from After colors, remove/keep, color_map,
shift(3,3), sorting rules, and identity.


tg_from_pairs(+Pairs, -Candidates)

Generate all candidates from a list of training pairs. Unions candidates
from each pair. Returns [identity] if Pairs is empty.


tg_filter_consistent(+Rules, +Pairs, -Consistent)

Filter a candidate list to rules that correctly transform every pair.
This is the core "search-and-filter" step.

Example:

Pairs = [[obj(r,[r(0,0)])]-[obj(g,[r(0,0)])]],
tg_filter_consistent([recolor(r,g), recolor(r,b)], Pairs, Consistent).
% Consistent = [recolor(r,g)]


tg_n_candidates(+Candidates, -N)

Count candidates in a list.


tg_all_scene_candidates(+Scene, +MaxShift, -Candidates)

All candidates from a single scene with a given shift bound. Equivalent to
taking the union of all generator predicates applied to the scene.


Typical Workflow: Enumerate then Filter

A complete "enumerate and filter" solver:

solve(TrainingPairs, TestInput, Output) :-
    TrainingPairs = [Before-_ | _],
    tg_all_scene_candidates(Before, 3, AllCands),
    tg_filter_consistent(AllCands, TrainingPairs, Consistent),
    Consistent = [BestRule | _],      % pick first consistent rule
    tg_apply_(BestRule, TestInput, Output).

Or use tg_from_pairs for pair-specific candidates:

solve2(TrainingPairs, TestInput, Output) :-
    tg_from_pairs(TrainingPairs, Cands),
    tg_filter_consistent(Cands, TrainingPairs, Consistent),
    Consistent = [Rule | _],
    tg_apply_(Rule, TestInput, Output).


Dependencies

None. Self-contained internal rule application engine. Uses library(lists)
and library(apply).

Summary

The transformgen pack is the exhaustive enumeration layer. It generates every
plausible rule from the scene data and hands them to a filter (tg_filter_consistent)
or ranker (scenerank, gridsolve) for selection. Combined with gridsolve, it
ensures that the correct rule is always in the candidate set, even if heuristic
inference would miss it.
