Chapter 265: Wave 47 - shape_count (WP-305, Layer 280)

shape_count solves ARC-AGI-2 task 58490d8a. The input grid contains two
regions: a rectangular template (all 0-cells with a single color marker at
odd rows) and a game board (non-zero background with colored shapes). For
each color named in the template, the rule counts how many non-overlapping
instances of that color-shape appear in the game board and writes the count
as repeated color markers in the output (which matches the template region).


PATTERN

The template occupies a rectangular block of 0-cells. Inside the template,
every even template row (0, 2, 4, ...) is all zeros. Every odd template row
(1, 3, 5, ...) has exactly one non-zero value at template column 1; that value
names a color to count.

The game board contains identical instances of small geometric shapes, one
shape type per color. Each instance is a fixed pattern of colored cells spread
across a tight bounding box. The count in the output is how many separate
(non-overlapping) instances appear.

Output: a copy of the template layout where each color marker is repeated
Count times at template columns 1, 3, 5, ..., 2*Count-1 in its row.


INSTANCE DETECTION: BBOX EXPANSION

Key challenge: some shape instances are NOT fully 4-connected internally.
Example from P1: the 8-shape is a 5-cell diamond-cross (C _ C / _ C _ / C _ C)
whose cells are only diagonally adjacent. Standard flood-fill would treat the
5 cells as 5 separate components.

The correct approach uses iterative bounding-box expansion:

  EXPAND(Seed, AllCells):
    Cur = {Seed}
    LOOP:
      (R0,R1,C0,C1) = tight_bbox(Cur)
      Expanded = {p in AllCells : R0-1 <= p.r <= R1+1, C0-1 <= p.c <= C1+1}
      IF Expanded == Cur: RETURN Cur  // stable - instance complete
      Cur = Expanded

This grows the bounding box by 1 in every direction and pulls in any same-color
cells nearby. The loop always terminates because Expanded is a superset of Cur
and bounded by the finite AllCells pool.

After extracting one instance, remove it from the pool and repeat with the
next top-left remaining cell. Count the iterations.

KEY PREDICATES

  arc2_sc_tmpl_bbox_(+Grid, -TR0, -TR1, -TC0, -TC1)
    Scans all cells for 0-values and returns min/max row and col.

  arc2_sc_tmpl_colors_(+Grid, +TR0, +TR1, +TC0, +TC1, -Colors)
    Scans odd template rows (1, 3, 5, ...) for the first non-zero value.
    Returns [(TR, Color)] list.

  arc2_sc_count_(+Grid, +Color, +TR0, +TR1, +TC0, +TC1, -Count)
    Gathers all Color cells outside template, sorts them, then calls
    arc2_sc_instances_ to count by iterative extraction.

  arc2_sc_expand_(+CurInst, +AllCells, -Inst)
    Iterative bbox expansion: tight bbox -> expand by 1 -> collect -> repeat.
    Returns stable set of cells forming one shape instance.

  arc2_sc_instances_(+SortedCells, +N0, -N)
    Recursively extracts instances from SortedCells, counting each one.

  arc2_sc_place_(+OutIn, +TR, +Color, +Count, +TC, +NTC, -Out)
    Places Count copies of Color at cols TC, TC+2, ... in row TR of output.

  arc2_transform(shape_count, +Grid, -Out)
    Orchestrator: tmpl_bbox -> tmpl_colors -> zeros -> fill_all
    (gather -> instances -> place for each color) -> Out.
