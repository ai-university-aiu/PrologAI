Chapter 82: Straight-Line Segment Detection and Drawing - The Line Pack

The line pack (Layer 100, Work Package 121) provides 14 predicates for working
with straight-line segments in 2D integer-coordinate grids. It covers cell
enumeration on horizontal, vertical, and 45-degree diagonal lines; grid drawing;
alignment tests; collinearity; endpoint extraction; interior gap queries; and
segment type classification.

All predicates use the li_* prefix.


82.1 What Problem Does the Line Pack Solve?

Many visual reasoning tasks require detecting which cells lie on a straight line,
or drawing a line of a given color across a grid. The line pack answers questions
like: What cells are on the horizontal segment from (2,1) to (2,5)? Is cell
(3,3) on the same diagonal as (1,1)? What cells lie strictly between (0,0) and
(3,3) on that diagonal? It also provides the reverse operation: given a grid,
draw a line segment onto it.


82.2 Segment Enumeration

The three enumeration predicates list every cell on a segment.

li_hline(+R, +C1, +C2, -Cells) lists cells on the horizontal segment at row R
from column C1 to column C2.

Example:

?- li_hline(1, 0, 3, Cells).
Cells = [1-0, 1-1, 1-2, 1-3]

?- li_hline(0, 5, 2, Cells).
Cells = [0-2, 0-3, 0-4, 0-5]

The columns are always returned in ascending order, regardless of whether C1 is
smaller or larger than C2.

li_vline(+C, +R1, +R2, -Cells) lists cells on the vertical segment at column C
from row R1 to row R2.

Example:

?- li_vline(2, 0, 3, Cells).
Cells = [0-2, 1-2, 2-2, 3-2]

Rows are also returned in ascending order.

li_diag_seg(+R1, +C1, +R2, +C2, -Cells) lists cells on the 45-degree diagonal
from (R1,C1) to (R2,C2). The path length must equal the column span:
abs(R2-R1) must equal abs(C2-C1). Cells are returned in walk order from the
start to the end.

Example:

?- li_diag_seg(0, 0, 2, 2, Cells).
Cells = [0-0, 1-1, 2-2]

?- li_diag_seg(2, 0, 0, 2, Cells).
Cells = [2-0, 1-1, 0-2]


82.3 Grid Drawing

The drawing predicates return a modified grid with a line segment painted to a
given color. All row and column indices are 0-based.

li_draw_h(+Grid, +R, +C1, +C2, +Color, -Result) draws a horizontal line.
li_draw_v(+Grid, +C, +R1, +R2, +Color, -Result) draws a vertical line.
li_draw_diag(+Grid, +R1, +C1, +R2, +C2, +Color, -Result) draws a 45-degree
diagonal line.

Example:

?- Grid = [[0,0,0],[0,0,0],[0,0,0]],
   li_draw_h(Grid, 1, 0, 2, 5, Result).
Result = [[0,0,0],[5,5,5],[0,0,0]]

?- Grid = [[0,0,0],[0,0,0],[0,0,0]],
   li_draw_v(Grid, 1, 0, 2, 4, Result).
Result = [[0,4,0],[0,4,0],[0,4,0]]

?- Grid = [[0,0,0],[0,0,0],[0,0,0]],
   li_draw_diag(Grid, 0, 0, 2, 2, 1, Result).
Result = [[1,0,0],[0,1,0],[0,0,1]]


82.4 Alignment Tests

The alignment predicates test whether two cells share a row, column, diagonal,
or anti-diagonal. They succeed or fail; they produce no output terms.

li_same_row(+R1, +C1, +R2, +C2) - cells share a row if R1 =:= R2.
li_same_col(+R1, +C1, +R2, +C2) - cells share a column if C1 =:= C2.
li_same_diag(+R1, +C1, +R2, +C2) - cells share a diagonal if C1 - R1 =:= C2 - R2.
li_same_anti(+R1, +C1, +R2, +C2) - cells share an anti-diagonal if R1 + C1 =:= R2 + C2.

Example:

?- li_same_diag(0, 0, 2, 2).
true.

?- li_same_anti(0, 4, 2, 2).
true.

?- li_same_row(1, 3, 2, 3).
false.


82.5 Collinearity Test

li_collinear(+R1, +C1, +R2, +C2, +R3, +C3) succeeds if three cells are
collinear: they all lie on the same row, the same column, the same diagonal
(C - R constant), or the same anti-diagonal (R + C constant).

Example:

?- li_collinear(2, 0, 2, 3, 2, 7).
true.

?- li_collinear(0, 0, 1, 1, 3, 3).
true.

?- li_collinear(0, 0, 1, 2, 2, 2).
false.


82.6 Endpoints

li_endpoints(+Cells, -First, -Last) extracts the first and last R-C pair from
a non-empty list of cells. This is useful when you have a segment computed by
li_hline, li_vline, or li_diag_seg and need only the boundary cells.

Example:

?- li_endpoints([0-0, 1-1, 2-2], First, Last).
First = 0-0, Last = 2-2.

?- li_endpoints([3-5], First, Last).
First = 3-5, Last = 3-5.


82.7 Interior Gap

li_gap(+R1, +C1, +R2, +C2, -Gap) returns the cells strictly between two
aligned points, excluding both endpoints. The two input cells must be on the
same row, column, diagonal, or anti-diagonal. If they are adjacent, Gap = [].

Example:

?- li_gap(1, 0, 1, 4, Gap).
Gap = [1-1, 1-2, 1-3].

?- li_gap(0, 2, 3, 2, Gap).
Gap = [1-2, 2-2].

?- li_gap(0, 0, 0, 1, Gap).
Gap = [].


82.8 Segment Type Classification

li_line_type(+R1, +C1, +R2, +C2, -Type) classifies the segment from (R1,C1)
to (R2,C2). Type is one of: point (same cell), horizontal, vertical, diagonal,
anti_diagonal, or other.

Example:

?- li_line_type(2, 0, 2, 5, Type).
Type = horizontal.

?- li_line_type(0, 0, 3, 3, Type).
Type = diagonal.

?- li_line_type(4, 4, 4, 4, Type).
Type = point.


82.9 Implementation Notes

Determinism is achieved through three strategies. Enumeration predicates
(li_hline, li_vline) use maplist/3 with unification lambdas of the form
[X, RC]>>(RC = R-X) rather than [X, R-X]>>true; the explicit unification in
the lambda body commits the result without leaving a choicepoint.

li_diag_walk_ and li_fill_cells_ use if-then-else instead of two-clause
recursion, so Prolog does not retain a choicepoint after reaching the base case.

li_collinear and li_line_type use if-then-else chains; the -> commit operator
prevents backtracking into later alternatives once a condition is satisfied.

li_replace_nth_ and li_last_ carry cuts on their base cases for the same reason.

Built-ins used without import: length/2.
Imported from library(lists): nth0/3, numlist/3, append/2.
Imported from library(apply): maplist/2, maplist/3.


82.10 Summary

The line pack adds Layer 100 to the PrologAI substrate with 14 li_* predicates
covering the full lifecycle of straight-line segments: enumeration, drawing,
alignment testing, collinearity, endpoint extraction, gap queries, and type
