plus_mark solves ARC-AGI-2 task 1818057f. The input grid contains cells of
value 4 on a background of value 2. Some subsets of 4-cells form a plus-shape:
a center cell with its four orthogonal neighbors (north, south, east, west)
each exactly one step away, all five cells holding value 4. The rule recolours
every cell that belongs to any such plus-shape to value 8. All other cells
remain unchanged.


Algorithm:

  1. H = grid height, W = grid width.
  2. PlusCells = {}: the set of cells to recolour.
  3. For each interior (R, C) with R in 1..H-2 and C in 1..W-2:
     If Grid[R][C] = 4 AND Grid[R-1][C] = 4 AND Grid[R+1][C] = 4
        AND Grid[R][C-1] = 4 AND Grid[R][C+1] = 4:
       Add (R,C), (R-1,C), (R+1,C), (R,C-1), (R,C+1) to PlusCells.
  4. Output = copy of Grid with PlusCells[i][j] set to 8.

Key predicates:

arc2_pm_is_plus_center_(+Grid, +H, +W, +R, +C):
  Bounds-check R and C as interior (not on boundary); nth0 lookups for center
  row and neighbor rows; all five cells must hold value 4.

arc2_pm_plus_cells_(+Grid, +H, +W, -PlusCells):
  between/3 enumeration over all candidate center positions inside findall;
  for each valid center, generate all 5 member cells; sort deduplicates.

arc2_transform(plus_mark, +Grid, -Out):
  Measures H and W; calls arc2_pm_plus_cells_; numlist for index lists;
  maplist builds each output row using memberchk to select 8 vs original value.

Key insight:

A plus center is uniquely identified by checking its four orthogonal
neighbors. Using between/3 inside findall ensures the candidate coordinates
are ground before the > comparison in arc2_pm_is_plus_center_ is called,
avoiding uninstantiated-variable errors.
