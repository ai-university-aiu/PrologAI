Chapter 288: Wave 70 - shape_catalog (WP-328, Layer 303)

shape_catalog solves ARC-AGI-2 task 67e490f4. Each 30x30 input grid
is dominated by one large lattice block built from exactly two
colors: a frame color (for example color 4) drawn over the background
color (for example color 1). The lattice encloses small pockets of
background cells shaped like 2x2 squares, dominoes, pluses,
L-triominoes, bars, and larger glyphs. The field outside the block is
scattered with small monochrome shapes, each drawn in its own color:
a shape catalogue. The task is to output the block alone, cropped
along its bounding box, with every background pocket recolored to the
color of the same-shaped catalogue entry. The training blocks are
11x11 and 13x13; the hidden test block is a 7x30 band spanning the
full grid width.

Rule: the outside shapes are a color dictionary keyed by shape. Every
pocket inside the block looks itself up in that dictionary - up to
rotation and reflection - and paints itself with the dominant color
it finds there.

Algorithm:
(1) Block detection: the background is the most frequent cell value
    (arc2_bg_color_/2); all non-background cells are partitioned
    into 4-connected same-color components (arc2_all_comps_/3); the
    component with the most cells is the frame lattice
    (scg_largest_/2). Every cell inside the lattice's bounding box
    must be frame color or background, and the block is cropped
    along that box (scg_bbox_/5, scg_crop_/6).
(2) Catalogue canonicalization: every non-frame component is a
    catalogue entry; its cells are shifted so the minimum corner
    sits at the origin (scg_norm_/2), the eight dihedral-four
    variants are generated by the eight scg_d4c_/6 coordinate maps
    (identity, two flips, half turn, transpose, both quarter turns,
    anti-transpose), and the lexicographically smallest variant is
    the canonical shape, paired with the entry's color
    (scg_canon_/2, scg_d4_/5).
(3) Motif extraction: the background pockets inside the block are
    recovered by rerunning arc2_all_comps_/3 over the cropped block
    with the frame color passed as the background argument - the
    pockets emerge as 4-connected motif components with no dedicated
    pocket-walking code.
(4) Dominant vote and paint: each motif is canonicalized the same
    way and collects the colors of every same-shaped catalogue entry
    (scg_assign_/3); at least one vote is required; the most
    frequent color wins, smallest color value on ties
    (scg_dominant_/2); the block is rebuilt with frame cells kept
    and motif cells recolored (scg_paint_/4).
(5) Induction pre-filter: the first output must fit inside the first
    input and be strictly smaller in at least one dimension, and the
    first input must carry at least five distinct colors; then all
    training pairs are verified with forall.

Two lessons make the rule honest. First, matching must be invariant
under the dihedral-four group: the test block holds a horizontal
domino whose only catalogue mates are drawn vertically, and the block
corners repeat one L-triomino in four orientations while the
catalogue draws it in fewer - raw normalized cell-set matching
fails. Second, the assignment is a vote, not a first match: in
training pair 1 the 2x2 square motif matches catalogue squares of
colors 8, 2, and 3 at the same time, and the correct output paints
it with color 8, the color holding the most catalogue entries. A
structural bonus: the two-color bounding-box guard already
guarantees that every other component stands outside the block, so
the catalogue is simply every non-frame component with no positional
filtering.

Key predicates:
  scg_largest_/2   - component with the most cells (frame lattice)
  scg_bbox_/5      - bounding box of a cell list
  scg_crop_/6      - cut the bounding-box subgrid out of the input
  scg_canon_/2     - canonical shape under the eight D4 maps
  scg_norm_/2      - shift cells so the minimum corner is the origin
  scg_d4_/5        - apply one dihedral-four transform to a shape
  scg_d4c_/6       - the eight per-cell coordinate maps
  scg_assign_/3    - motif-to-color assignment by canonical shape
  scg_dominant_/2  - most frequent vote, smallest color on ties
  scg_paint_/4     - rebuild the recolored block
  arc2_transform(shape_catalog, +Grid, -Out)
  arc2_induce_rule(+Pairs, shape_catalog)
