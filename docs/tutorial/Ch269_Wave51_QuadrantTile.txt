quadrant_tile solves ARC-AGI-2 task f931b4a8. The input grid is divided into
four quadrants at its midpoint. The top two quadrants encode output dimensions
by non-zero cell count. The bottom-left quadrant is the fill template. The
bottom-right quadrant is the tile template with zeros marking fill positions.

The output is produced by periodically tiling the bottom-right template over
the computed output dimensions, filling each zero from the bottom-left template
using the tile-block row index (not the row within the tile) for the BL row
lookup.


Algorithm:

  1. R = grid height, C = grid width. SplitR = R // 2, SplitC = C // 2.
  2. BL = Grid[SplitR..R-1][0..SplitC-1] (fill template).
  3. BR = Grid[SplitR..R-1][SplitC..C-1] (tile template).
  4. Out_H = count of non-zero cells in Grid[0..SplitR-1][0..SplitC-1].
  5. Out_W = count of non-zero cells in Grid[0..SplitR-1][SplitC..C-1].
  6. For output (R,C): tile_r = R mod BRh, tile_c = C mod BRw, ti = R // BRh.
     If BR[tile_r][tile_c] != 0: use that value.
     Else: use BL[ti mod BLh][tile_c mod BLw].
