Chapter 149: Objcomp Pack (oc_*, Layer 167)

The objcomp pack (oc_* prefix, Layer 167) provides 14 predicates for adjacency
and connected-component analysis over collections of obj(Color, Cells) terms.
Two objects "touch" when any cell of the first is 4-connected (same row with
adjacent columns, or same column with adjacent rows) to any cell of the second.

149.1 Loading

:- use_module(pack(objcomp)).

149.2 The Touching Relation

object_component_touches(+Obj1, +Obj2): true when any cell of Obj1 is 4-adjacent to any
cell of Obj2. Cuts after finding the first adjacent cell pair.

    ?- object_component_touches(obj(a,[r(0,0)]), obj(b,[r(0,1)])).
    true.

    ?- object_component_touches(obj(a,[r(0,0)]), obj(b,[r(0,3)])).
    false.

149.3 Pair and Adjacency Queries

object_component_touching_pairs(+Objs, -Pairs): list of O1-O2 pairs where both objects are
in Objs and they touch. Each unordered pair appears once.

    ?- Objs = [obj(a,[r(0,0)]), obj(b,[r(0,1)]), obj(c,[r(2,2)])],
       object_component_touching_pairs(Objs, Pairs).
    Pairs = [obj(a,...)-obj(b,...)].  % only a-b touch

object_component_adj_list(+Objs, -AdjList): Obj-[Neighbors] pairs for every object.

object_component_degree(+Obj, +Objs, -N): count of touching neighbors.

    ?- object_component_degree(obj(a,[r(0,0)]), [obj(a,[r(0,0)]),obj(b,[r(0,1)]),obj(c,[r(2,2)])], D).
    D = 1.  % a touches only b

149.4 Isolation

object_component_isolated(+Objs, -Isolated): objects that do not touch any other in Objs.

    ?- object_component_isolated([obj(a,[r(0,0)]),obj(b,[r(0,1)]),obj(c,[r(5,5)])], I).
    I = [obj(c,[r(5,5)])].

149.5 Connected Components

object_component_connected(+Obj, +Objs, -Component): all objects reachable from Obj through
the touching relation, including Obj itself. Uses BFS.

    % A-B touch, B-F touch, F-C touch. D touches A. E is isolated.
    % object_component_connected(A, [A,B,C,D,E,F], Comp) returns [A,B,C,D,F].

object_component_components(+Objs, -Components): all touching-connected components. Each
component is a sorted list. Useful when you need to know how objects cluster.

object_component_num_components(+Objs, -N): number of components.

    ?- object_component_num_components([obj(a,[r(0,0)]),obj(b,[r(0,1)]),obj(c,[r(5,5)])], N).
    N = 2.  % {a,b} and {c}

149.6 Component Selection

object_component_largest_component(+Objs, -Component): the component with the most objects.

object_component_smallest_component(+Objs, -Component): the component with the fewest objects.

object_component_singleton_components(+Objs, -Singles): components of size exactly 1.

object_component_shared_components(+Objs, -Groups): components of size 2 or more.

149.7 Degree Aggregates

object_component_max_degree(+Objs, -MaxDeg): maximum touch degree across all objects.

object_component_sort_by_degree(+Objs, -Sorted): objects sorted ascending by degree.
Objects with degree 0 (isolated) come first.

    ?- object_component_sort_by_degree(Objs, [First|_]),
       object_component_degree(First, Objs, D).
    D = 0.  % isolated objects come first

149.8 Predicate Reference

Touching:
  object_component_touches(+Obj1, +Obj2)

Pairs and adjacency:
  object_component_touching_pairs(+Objs, -Pairs)
  object_component_adj_list(+Objs, -AdjList)
  object_component_degree(+Obj, +Objs, -N)

Isolation:
  object_component_isolated(+Objs, -Isolated)

Connected components:
  object_component_connected(+Obj, +Objs, -Component)
  object_component_components(+Objs, -Components)
  object_component_num_components(+Objs, -N)

Component selection:
  object_component_largest_component(+Objs, -Component)
  object_component_smallest_component(+Objs, -Component)
  object_component_singleton_components(+Objs, -Singles)
  object_component_shared_components(+Objs, -Groups)

Degree aggregates:
  object_component_max_degree(+Objs, -MaxDeg)
  object_component_sort_by_degree(+Objs, -Sorted)
