CHAPTER 15 - THE AGENCY PACK: THE AGENTIC EXECUTION LOOP

The agency pack gives PrologAI a formal, observable, and bounded loop for pursuing goals through sequences of reasoning and action steps.

This is the computational core of agentic behavior: the ability to observe a situation, reason about it, take an action, observe what happened, and repeat, all within a defined budget, under full observation, with a safe exit when things go wrong.


THE AGENTIC LOOP IN PLAIN TERMS

Think of an agent as a problem-solver that works in steps.

At each step, the agent looks at its current goal, thinks about what to do next, does it, and looks at what happened.

If the action moved it closer to the goal, it keeps going.

If it is stuck repeating the same thing, it can detect that and try something different.

If it cannot proceed without human help, it escalates.

If it runs out of budget, it stops cleanly.

All of this is what the agency pack provides.


LOADING THE AGENCY PACK

    :- use_module(library(ephemera)).
    :- use_module(library(agency)).

The agency pack depends on the ephemera pack, so both must be loaded.


THE LOOP LIFECYCLE

Every goal pursuit is tracked as a loop.

Step 1 - Create a loop:

    ?- ag_loop_create(my_goal, 20, LoopId).
    LoopId = ag_loop_1.

This allocates a unique loop identifier, stores the goal (my_goal) and budget (20 steps), and initializes the goal stack with my_goal as the first item.

Step 2 - Run the loop:

    ?- ag_loop_run(LoopId, my_reasoner, Outcome).

my_reasoner is the name of a Prolog predicate you write that decides what to do at each step.

It will be called as: call(my_reasoner, LoopId, CurrentGoal, Thought, Action).

Your reasoner produces a Thought (a term recording your reasoning) and an Action (a structured term describing what to do).

Step 3 - Inspect the outcome:

    ?- ag_loop_outcome(LoopId, Outcome).
    Outcome = done(success).

Step 4 - Inspect the trace:

    ?- ag_loop_trace(LoopId, Steps).


THE SEVEN ACTION TYPES

Your reasoner communicates what to do by unifying Action with one of these terms:

action_eval(GoalTerm)

Evaluates GoalTerm as a Prolog goal using the ephemera pack.

Useful for testing facts, calling predicates, or computing values.

    Action = action_eval(member(X, [1,2,3]))

action_shell(Argv, Timeout)

Runs a shell command and captures output.

    Action = action_shell(['python3', 'script.py'], 30)

action_push_goal(Subgoal)

Adds a new subgoal to the front of the goal stack.

Use this to decompose a complex goal into steps.

    Action = action_push_goal(fetch_data)

action_push_goals(Subgoals)

Adds a list of subgoals at once.

    Action = action_push_goals([fetch_data, validate_data, process_data])

action_mark_done(Result)

Declares the loop complete with a final result.

    Action = action_mark_done(all_tasks_complete)

action_escalate(Reason)

Requests human oversight. The loop stops safely.

    Action = action_escalate(stuck_need_credentials)

action_pop_goal

Removes the current subgoal from the stack (when it is finished).


WRITING A REASONER

A reasoner is a Prolog predicate that implements your strategy.

Here is a simple example that evaluates one arithmetic goal and then marks done:

    my_reasoner(LoopId, CurrentGoal, Thought, Action) :-
        ( CurrentGoal = compute(X, Y)
        ->  Thought = 'I will compute the sum',
            Action  = action_eval((Z is X + Y, nb_setval(result, Z)))
        ;   Thought = 'Goal complete',
            nb_getval(result, _),
            Action  = action_mark_done(success)
        ).

The reasoner can also call ag_detect_loop/2 to check for cycles:

    ?- ag_detect_loop(LoopId, IsLooping).
    IsLooping = false.

If IsLooping is true, the reasoner can escalate or change strategy.


THE GOAL STACK

The goal stack is a LIFO (Last In, First Out) ordered list of subgoals.

When the loop starts, the top-level goal is the only item on the stack.

As the loop runs, the reasoner can push subgoals to decompose the top-level goal into smaller tasks.

To push a subgoal manually:

    ?- ag_push_subgoal(LoopId, sub_task_a).

To pop the current subgoal:

    ?- ag_pop_subgoal(LoopId, Popped).
    Popped = sub_task_a.


BUDGET MANAGEMENT

Every loop has a step budget.

Each step costs one unit of budget.

To check how much budget remains:

    ?- ag_budget_remaining(LoopId, N).
    N = 15.

The budget is decremented automatically at the start of each step.

When the budget reaches zero, the loop stops with status budget_exhausted.

The budget floor is zero; it never goes negative.


LOOP DETECTION

If your reasoner keeps producing the same thought and the same action in consecutive steps, ag_detect_loop/2 returns true.

    ?- ag_detect_loop(LoopId, IsLooping).

Check this in your reasoner to avoid getting stuck.


ESCALATION

When the loop cannot proceed without human input, call ag_escalate:

    Action = action_escalate(need_password_for_database)

This records the reason in an escalation fact and marks the loop as escalated(Reason).

The loop is now terminal.

To check the outcome:

    ?- ag_loop_outcome(LoopId, escalated(Reason)).
    Reason = need_password_for_database.


READING THE TRACE

Every step is recorded.

    ?- ag_loop_trace(LoopId, Steps).

Each entry is a step(N, Thought, Action, Observation) term.

N is the step number, starting at 1.

Thought is whatever your reasoner produced.

Action is the action term.

Observation is what the action execution returned (obs_eval, obs_shell, obs_done, etc.).

The trace is the full, auditable record of everything the agent did and thought.


TAKING INDIVIDUAL STEPS

Instead of running the full loop, you can take one step at a time:

    ?- ag_loop_step(LoopId, my_reasoner, StepResult).
    StepResult = step_ok(Thought, Action, Observation).

This gives you fine-grained control, useful for debugging or interactive sessions.


SUMMARY

ag_loop_create/3      Allocate a loop with a goal and step budget.
ag_loop_run/3         Run the loop to completion.
ag_loop_step/3        Execute a single step.
ag_loop_done/1        Check whether the loop has reached a terminal state.
ag_loop_outcome/2     Retrieve the final status.
ag_loop_trace/2       Retrieve the full step-by-step trace.
ag_push_subgoal/2     Add a subgoal to the goal stack.
ag_pop_subgoal/2      Remove the top subgoal.
ag_budget_remaining/2 Query remaining steps.
ag_budget_decrement/1 Consume one step (called automatically each step).
ag_detect_loop/2      Detect repeated thought-action pairs.
ag_escalate/2         Record escalation and halt the loop safely.
ag_step_record/4      Record one step manually.
