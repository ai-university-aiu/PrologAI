PrologAI Data Layer Pack Reference
===================================

This file contains the full predicate-level descriptions for each pack in the
Data Layer section of PrologAI. The README.md contains short titles only.

vector_backend
The Vector Backend

    a backend-agnostic six-predicate interface. Ships with two backends: the built-in pure-Prolog fallback and the RuVector HNSW (Hierarchical Navigable Small World) + SIMD HTTP REST backend (port 6333). Any vector store plugs in without changing the reasoning layer. Run `vector_backend_set_backend(ruvector)` to switch; use `run_bakeoff([prolog, ruvector], [100, 1000])` to compare. Includes in-process shadow store (`backend_ruvector_shadow/5`) and rebuild predicate (`vector_backend_rebuild/1`) — if the RuVector server restarts, call `vector_backend_rebuild(Ref)` to re-insert all shadow vectors and restore the HNSW index. Bakeoff result: ruvector scores 0.2655 vs. prolog 0.0650 at 100,000 vectors (4x advantage, HNSW sub-linear scaling confirmed).

vector_symbolic_architecture
Compositional Vector Binding (VSA — Vector Symbolic Architecture)

    MAP and HRR algebras. Concepts are built by binding, not by lookup.

lattice_cryptography
Lattice Cryptographic Privacy Layer

    RSA, ECDH, and post-quantum (PQC) hybrid encryption for sensitive Lattice data.

ephemera
Ephemeral Code Synthesis, Execution, and Skill Persistence (PR 53)

    compose short-lived programs in Prolog, Python, or Bash, run them with a wall-clock timeout, capture stdout and stderr, and log execution traces. Useful ephemera can be named, saved, indexed, retrieved by name, and re-run as skills without re-synthesis. Key predicates: `ephemera_eval/3` (Prolog goal with timeout), `ephemera_shell/3` (shell command), `ephemera_ephemeral/4` (language-specific script), `ephemera_iterate/5` (synthesize-execute-check loop), `ephemera_skill_save/4`, `ephemera_skill_run/3`, `ephemera_skill_list/1`.

agency
Agentic Execution Loop (PR 54)

    formal, observable, bounded goal-pursuit loop. Allocate a loop with a step budget, supply a reasoning goal, and let the loop execute Observe-Reason-Act-Observe steps until done, budget-exhausted, or escalated to human oversight. Full trace recording, goal stack management, loop detection, and safe escalation. Key predicates: `agency_loop_create/3`, `agency_loop_run/3`, `agency_detect_loop/2`, `agency_escalate/2`, `agency_loop_trace/2`.

refinery
Evaluator-Optimizer and Metacognitive Quality Layer (PR 55)

    critique outputs against named criteria, score them as a fraction in [0.0, 1.0], drive iterative improvement cycles with an improver goal, run full evaluator-optimizer loops, explore multiple reasoning paths and rank by score, and maintain a lesson database that records what went wrong and recalls it before future attempts. Key predicates: `refinery_critique/4`, `refinery_score/3`, `refinery_optimize/5`, `refinery_explore_paths/4`, `refinery_learn/3`.

grid
ARC-AGI Grid Perception and Manipulation (PR 56)

    native grid operations for the ARC-AGI benchmark: dimensions (`grid_size/3`), zero-based cell access (`grid_cell/4`, `grid_row/3`, `grid_col/3`), color analysis (`grid_colors/2`, `grid_color_count/3`, `grid_color_map/3`), 4-connected object extraction (`grid_objects/3`, `grid_connected/3`), bounding box (`grid_bounding_box/3`), seven spatial transformations (rotate 90/180/270, reflect h/v/d1/d2), translation, cropping, overlay compositing, diff, symmetry detection (`grid_symmetry/2`), flood fill (`grid_fill/5`), uniform grid construction (`grid_make/4`), and single-cell mutation (`grid_set_cell/5`). The perceptual foundation for ARC-AGI solving.

analogy
Analogy (analogy_*, Layer 394, WP-419; converged with the ARC grid-analogy pack)
    Relational structure-mapping half —
    analogy_objects(+Relations,-Objects): the distinct objects a relation set mentions (rel(Type,A,B)).
    analogy_map_object(+Mapping,+Object,-Image): apply an object mapping (list of Src-Tgt pairs).
    analogy_preserved(+Mapping,+Source,+Target,-Count): how many source relations line up in the target.
    analogy_structure_map(+Source,+Target,-Mapping,-Score): the injective object mapping that preserves the most relations.
    analogy_transfer(+Mapping,+Term,-Image): carry a term (a rule, goal, or disposition) from source objects to their target images.
    Grid-analogy half (absorbed from the ARC pack) —
    analogy_solve_from_examples(+Examples,+Input,-Output): infer the transform from (input,output) train pairs and apply it.
    analogy_examples_isometry(+Examples,-Iso,-ColorMap) / analogy_apply_isometry(+Iso,+Grid,-Grid2): the D4 dihedral group (identity, rot90/180/270, ref_h/v/d1/d2).
    analogy_grid_isometry(+A,+B,-Iso) / analogy_isometry_candidates(+A,+B,-Isos): grids equal up to isometry.
    analogy_infer_color_map(+A,+B,-Map) / analogy_apply_color_map(+Map,+Grid,-Grid2) / analogy_grid_color_map(+A,+B,-Map): colour permutations.
    analogy_normalize_shape(+Shape,-Norm) / analogy_shape_equal(+A,+B) / analogy_shape_isometry(+A,+B,-Iso): shape equality up to isometry.
    analogy_shape_bbox(+Shape,-BBox) / analogy_shape_size(+Shape,-N) / analogy_match_objects(+A,+B,-Pairs) / analogy_object_offset(+A,+B,-Offset): shape and object geometry.
